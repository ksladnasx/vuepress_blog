网络智能技术 期末复习笔记

一、认证技术

认证技术主要包括对用户身份的认证和对消息的认证两种。

实体认证用于验证信息发送者是否真实，防止冒充；

消息认证则用于验证信息的完整性，确保数据在传输或存储过程中未被篡改、重放或延迟。

1. 双重签名

双重签名是为解决电子交易中三方之间信息传输的安全性问题而设计的，具体做法是：发送方分别用Hash函数对两个信息A和B进行处理，生成各自的数字摘要A和B，然后将二者合并形成一个新的信息C，再用Hash函数处理得到数字摘要C，最后用发送方的私人密钥对双重数字摘要进行加密，即得到双重数字签名。

2. 数字时间戳

数字时间戳是对电子文件签署的日期和时间进行安全性保护和有效证明的技术，必须由专门的数字时间戳服务中心添加。

它实际上是一个经过加密后形成的凭证文档，一般包括三个部分：信息的数字摘要、数字时间戳服务中心收到数字摘要的时间和日期、以及数字时间戳服务中心的数字签名。

3. 静态口令与动态口令

静态口令由用户设定静态密码，计算机进行验证。

在技术层面，口令以密文方式存储，传输时通常采用CS模式，通过加密口令或散列函数运算后传输。

然而，静态口令存在诸多安全问题：静态传输过程容易被截获，系统中用户口令以文件形式存储易被攻击者获取，无法抵御重放攻击，且只能进行单向认证。

动态口令则根据专门算法生成一个不可预测的6位或8位随机数字组合，每个密码一次有效，被公认为最安全的身份认证方案。其登录形式通常包括用户名、静态密码和动态密码三项。

4. 数字证书

数字证书是由权威机构——CA证书授权中心发行的，能提供在Internet上进行身份验证的一种权威性电子文档，人们可以在互联网交往中用它来证明自己的身份和识别对方的身份。

二、防火墙原理与技术

1. 防火墙的概念与基本原则

防火墙是建立在内外网络边界上的过滤封锁机制，内部网络被认为是安全和可信赖的，而外部网络（通常是Internet）被认为是不安全和不可信赖的。防火墙的作用是防止不希望的、未经授权的通信进出被保护的内部网络，通过边界控制强化内部网络的安全政策。

防火墙必须遵循三个基本原则：

所有进出被保护网络的通信必须通过防火墙；

所有通过防火墙的通信必须经过安全策略的过滤或者防火墙的授权；

防火墙本身是不可被侵入的。

2. 防火墙的附加功能

除了基本的访问控制，防火墙还具备以下附加功能：

流量控制，可针对不同用户限制不同的流量，合理使用带宽资源；

NAT（网络地址转换），通过修改数据包的源地址或目的地址来节省IP地址资源并隐藏内部IP地址；

VPN（虚拟专用网），利用数据封装和加密技术，使本只能在私有网络上传送的数据能够通过公共网络进行传输，大大降低系统费用。

3. 防火墙的局限性

安装防火墙并不能做到绝对的安全，它有许多防范不到的地方。

首先，防火墙不能防范不经由防火墙的攻击，例如如果允许从受保护网内部不受限制地向外拨号，用户可能形成与Internet的直接连接从而绕过防火墙。

其次，防火墙不能防止感染了病毒的软件或文件的传输，这只能在每台主机上安装反病毒软件来解决，因为病毒类型太多，操作系统也有多种，不能期望防火墙对每一个进出内部网络的文件进行扫描。

此外，防火墙不能防止数据驱动式攻击，有些表面无害的数据通过电子邮件发送或其他方式复制到内部主机上，一旦被执行就形成攻击。

最后，防火墙不能防范恶意的内部人员侵入，因为内部人员通晓内部网络的结构，其内部入侵行为可能不经过防火墙。

4. 包过滤技术与代理技术的比较

包过滤技术和应用网关通过特定的逻辑判断来决定是否允许特定的数据通过，其优点是速度快、实现方便，缺点是审计功能差，过滤规则的设计存在矛盾：规则简单则安全性差，规则复杂则管理困难。一旦判断条件满足，防火墙内部网络的结构和运行状态便暴露在外用户面前。

代理技术则能进行安全控制和加速访问，有效地实现防火墙内外计算机系统的隔离，安全性好，并能实施较强的数据流监控、过滤、记录和报告等功能。其缺点是每一种应用服务都必须为其设计一个代理进行安全控制，而每一种网络应用各不相同，分析困难，因此实现也较为复杂。

三、入侵检测系统（IDS）

1. 基本概念

入侵检测是最近十余年发展起来的一种动态的监控、预防或抵御系统入侵行为的安全机制，主要通过监控网络、系统的状态、行为以及系统的使用情况，来检测系统用户的越权使用以及系统外部的入侵者利用系统的安全缺陷对系统进行入侵的企图。进行入侵检测的软件与硬件的组合便是入侵检测系统（IDS）。

2. 入侵检测系统的分类

根据系统各个模块运行的分布方式，可分为集中式入侵检测系统（各个模块包括数据的收集与分析以及响应都集中在一台主机上运行，适用于网络环境简单的情况）和分布式入侵检测系统（各个模块分布在网络中不同的计算机、设备上，分布性主要体现在数据收集模块上，当网络环境复杂、数据量大时，数据分析模块也会分布，一般按照层次性原则组织）。

根据检测系统对入侵攻击的响应方式，可分为主动的入侵检测系统（检测出入侵后自动对目标系统中的漏洞采取修补、强制可疑用户退出系统以及关闭相关服务等对策）和被动的入侵检测系统（检测出入侵后仅产生报警信息通知系统安全管理员，后续处理由管理员完成）。

根据目标系统的类型，可分为基于主机的入侵检测系统（监测系统、事件和操作系统下的安全记录以及系统记录，当文件发生变化时与攻击标记相比较）和基于网络的入侵检测系统（使用原始网络数据包作为数据源，利用运行在混杂模式下的网络适配器实时监视并分析通过网络的所有通信业务）。

3. 入侵检测的优缺点

优点：提高信息安全构造的其他部分的完整性，提高系统的监控能力，从入口点到出口点跟踪用户的活动，识别和汇报数据文件的变化，侦测系统配置错误并纠正它们，识别特殊攻击类型并向管理人员发出警报进行防御。

缺点：不能弥补差的认证机制，如果没有人的干预不能管理攻击调查，不能知道安全策略的内容，不能弥补网络协议上的弱点，不能弥补系统提供质量或完整性的问题，不能分析一个堵塞的网络。

4. 基于完整性检查的入侵检测技术（散列算法）

常用的散列算法有MD5、CRC16、CRC32、Snefru、MD4、MD2、SHA和Haval等。散列算法通常实现了将任意长度的消息m压缩成一固定长度的散列值h，通过对散列值的校验能检测到对消息m的篡改、抵赖或伪造。它具有以下特性：易用性（对任意长度的m，计算h=H(m)很容易）；单向性（给定h，计算m使得m=F(h)很困难）；无碰撞性（给定m，要找到另一个消息m'满足H(m')=H(m)很困难，这保证了对原文有改动时很难使文件内容的散列值保持不变）。

5. 入侵检测技术现状与问题

目前的入侵检测产品大多存在以下矛盾：误报和漏报的矛盾，隐私和安全的矛盾，被动分析与主动发现的矛盾，海量信息与分析代价的矛盾，功能性和可管理性的矛盾，单一的产品与复杂的网络应用的矛盾。

6. 入侵防御系统（IPS）的提出

有学者建议使用全新的防御系统IPS来代替IDS，这是因为IDS存在如下问题：不能解析加密数据流，也就不能检测加密流量下的攻击；随着网络交换频率的增大，IDS只能监视到少量的数据流量；受硬件和软件的限制，IDS只能检测恶意攻击却不能有效阻止它们；误报与漏报现象严重。

四、无线网络基础

1. 移动通信代际发展

移动互联网指的是以各种类型的移动终端作为 接入设备,使用各种移动网络 作为接入网络,从而实现包括 传统移动通信、传统互联网及其各种融合创新服务的新型业务模式。

第一代移动通信（1G）主要解决语音通信的问题。第二代移动通信（2G）支持窄带的分组数据通信，最高理论速率为236kbps，其制式包括D-AMPS、GSM、CDMA以及日本的PDS。第三代移动通信（3G）在2G的基础上发展了图像、音乐、视频流等高带宽多媒体通信，并提高了语音通话安全性，解决了部分移动互联网相关网络及高速数据传输问题，最高理论速率为14.4Mbps。第四代移动通信（4G）是专为移动互联网而设计的通信技术，从网速、容量、稳定性上相比之前技术都有了跳跃性提升，传输速度可达100Mbit/s甚至更高。

2. CDMA技术特点

CDMA（码分多址）的国际标准是IS-95，其特点是每个站使用整个频段发送信号，多个站的信号可以线性叠加，利用编码技术分离并发的传输。接收端能提取出期望的信号，同时拒绝所有其它的信号，并把这些信号当作噪声。

3. 三种3G技术的比较

WCDMA基于GSM-MAP，采用FDD双工方式，双向信道带宽10MHz，码片速率3.84Mbps；CDMA2000基于ANSI-41，采用FDD双工方式，双向信道带宽2.5MHz，码片速率1.2288Mbps；TD-SCDMA基于GSM-MAP，采用TDD双工方式，双向信道带宽1.6MHz，码片速率1.28Mbps。在基站同步方面，WCDMA可异步（同步可选），CDMA2000和TD-SCDMA均为同步。

4. 无线网络基本概念与分类

无线网络是采用无线通信技术实现的网络，既包括允许用户建立远距离无线连接的全球语音和数据网络，也包括为近距离无线连接进行优化的红外线技术及射频技术，与有线网络的用途十分类似，最大的不同在于传输媒介的不同，利用无线电技术取代网线，可以和有线网络互为补充。

从无线网络覆盖范围看，可分为系统内部互连/无线个域网（WPAN）、无线局域网（WLAN）、无线城域网/广域网（WMAN/WWAN）。从应用角度看，还可以划分出无线传感器网络、无线Mesh网络等。

5. 无线个域网与体域网

无线个域网（WPAN）是一种在个人操作空间之内的设备进行通信和相互连接的通信网络，其特点是短距离、低功耗和低成本。蓝牙是一种典型的短距离无线网络，此外还有红外无线传输技术、家庭WiFi、Zigbee、超宽带等。

无线穿戴网络是基于短距离无线通信技术（蓝牙和ZigBee等）与可穿戴式计算机技术，穿戴在人体上、具有智能收集人体和周围环境信息的一种新型个体域网。体域网（BAN）是附着在人体上的一种网络，由一套小巧可移动、具有通信功能的传感器和身体主站（协调器）组成，每一传感器可穿戴在身上或植入体内。

6. 无线Mesh网络

无线Mesh网络（无线网状网络）是一种与传统无线网络完全不同的新型无线网络，由移动Ad Hoc网络演变而来，被形象称为无线版本的Internet。在无线Mesh网络中，任何无线设备节点都可以同时作为AP和路由器，每个节点都可以发送和接收信号，并与一个或多个对等节点进行直接通信。如果最近的AP由于流量过大而导致拥塞，数据可以自动重新路由到一个通信流量较小的邻近节点进行传输，直到到达最终目的地。

五、机器学习基础

1. 学习系统的基本结构与学习策略

机器学习系统的基本结构可建立为简单的学习模型。学习策略主要包括：机械学习（Rote Learning，即死记硬背式学习，把新知识存储起来供需要时检索调用，最简单的学习方法，不需要计算和推理）；归纳学习（包括有监督的示例学习和无监督的观察与发现学习）；类比学习（根据不同领域知识的相似性进行学习）；解释学习（结合演绎与归纳）；以及神经网络学习。

学习类型核心机制一句话例子
机械学习存储→检索背电话本，原样问原样答
归纳学习（监督）从带标签例子学规律老师标好猫/狗图片，学会分类
归纳学习（无监督）发现无标签数据的结构自己按形状或颜色分类积木
类比学习跨领域映射用“水流”理解“电流”
解释学习演绎+归纳，用理论解释例用公式解释三角形面积后，记住该数值条件
神经网络学习自动调整连接权重拧电炉旋钮多次试到水温正确

2. 监督学习

监督学习是利用已知类别的样本去训练算法从而调整分类器的参数，其任务是学习一个模型，使模型能够对任意给定的输入，对其相应的输出做出一个好的预测。常见的监督学习算法有：决策树、Adaboost算法、朴素贝叶斯算法、回归算法、支持向量机。

3. 模型选择与评估

奥卡姆剃刀原理指出，在所有可能的模型中，能够很好地解释已知数据并且十分简单的是最好的模型，也是应该选择的模型。

意思就是，优先选择假设最少、参数最少、复杂度最低且能解释数据的模型，避免不必要的复杂性（“剃掉”多余的假设）。

经验风险最小化策略认为经验风险最小的模型是最优模型，而结构风险最小化是为了防止过拟合而提出的策略，它在经验风险的基础上加上表示模型复杂度的正则化项或惩罚项，公式为$R(f)=1/N ΣL(y_i,F(x_i)) + λJ(F)$，目标是使R(f)最小化。

交叉验证是模型选择的重要方法。简单交叉验证是将数据随机分为训练集和测试集；S折交叉验证是将数据切分成s个互不相交的大小相同的子集，利用s-1个子集训练模型，用余下的子集测试，重复多次后选出平均测试误差最小的模型；留一交叉验证是当S=N时的情况，通常在数据缺乏时使用。如果数据充足，可将数据切分成训练集、验证集和测试集三部分，训练集用于训练模型，验证集用于模型选择，测试集用于最终评估。

4. 决策树学习

决策树学习由昆兰于1979年提出，是一种以信息熵的下降速度作为属性选择标准的学习算法，输入是一个用来描述各种已知类别的例子集，学习结果是一棵用于进行分类的决策树。其基本步骤为：令根结点包含例子集中所有实例；如果每个叶结点包含的例子都属于同一分类，则停止划分；否则对需要进一步划分的叶结点，找出对该结点例子集S来说信息熵最小的属性，根据该属性的值对S进行划分，每个值生成一个分枝，然后重复上述步骤。

5. 集成学习（元算法）

集成算法是将不同分类器组合起来的方法，可以是不同算法的集成，也可以是同一算法在不同设置下的集成，还可以是数据集不同部分分配给不同分类器之后的集成。

Adaboost算法具有以下特点：是一种有很高精度的分类器；可以使用各种方法构建子分类器，算法提供的是框架；当使用简单分类器时，计算出的结果容易理解，且弱分类器构造极其简单；简单，不用做特征筛选；不用担心过拟合问题。

Bagging方法（自举汇聚法）是在原始数据集中通过随机有放回地抽取样本（替换抽样）生成s个新数据集，每个数据集大小与原数据集相等。对每个数据集应用某个学习算法得到s个分类器，当对新数据进行分类时，应用这s个分类器进行投票，选择得票最多的类别作为最终分类结果。

6. 贝叶斯分类

贝叶斯分类是一类以贝叶斯定理为基础的分类算法。假设一个样本集的数据分为两类，P1(x,y)表示数据点属于类别1的概率，P2(x,y)表示属于类别2的概率，如果P1(x,y) > P2(x,y)则数据属于类别1，否则属于类别2。贝叶斯分类的优点是在数据较少的情况下仍然有效，可以处理多类别问题；缺点是对输入数据的准备方式比较敏感。

六、神经网络

1. 人工神经网络的定义与特点

人工神经网络是以对大脑的生理角度研究成果为基础，对动物或自然神经网络若干基本特性进行抽象和模拟，是由人工建立的以有向图为拓扑结构的动态系统，通过对连续或断续的输入作状态响应而进行信息处理。

人工神经网络具有以下突出优点：可以充分逼近任意复杂的非线性函数；具有分布式信息存贮特点，有很强的鲁棒性和容错性；具有巨量信息并行处理和大规模平行计算能力；具有自组织、自学习功能，是自适应组织系统。

2. 神经网络的结构

神经网络的结构按照神经元连接方式可分成前馈网络和反馈网络。前馈网络从输入层到输出层通过单向连接流通，只有前后相邻两层之间神经元相互全连接，从上一层接收信号输送给下一层神经元，同层神经元之间没有连接，各神经元之间也没有反馈，是一种静态的非线性映射，比较适用于模式识别、分类和预测评价问题。

反馈网络与前馈网络的区别主要在于神经元输出至少有一条反馈回路，信号可以正向或反向流通，由于反馈回路的存在，对神经网络的学习能力和决策都产生很大影响，需要一定的时间才能达到稳定。典型的反馈型网络有Hopfield网络、Boltzmann机网络、Kohonen网络。

3. MP神经元模型与激励函数

MP神经元模型将每一个神经元看作二进制阈值元件，输出为0或1，分别表示抑制或兴奋两种状态。常用的激励函数包括双曲正切函数f(X) = (1 - e^{-aX})/(1 + e^{-aX})（a>0）以及高斯函数f(X) = exp(-X^2/δ^2)。

4. BP神经网络

BP网络通常包含一个或多个隐含层，隐层中的神经元一般均采用S型函数作为激励函数，输出层神经元可以根据实际情况选择线性激励函数（此时整个网络的输出可以取任意值）或S型函数（此时整个网络的输出限定在较小范围内）。BP神经网络的输入与输出关系是一个高度非线性映射关系，如果输入结点数为m，输出结点数为n，则网络是从m维欧氏空间到n维欧氏空间的映射。

5. RBF神经网络

RBF神经网络由一个隐含层和一个输出层组成，结构与一般前向网络相似，同层神经元之间没有连接，相邻两层的神经元完全连接。隐层节点通过选取的激励函数（RBF激励函数）实现输入、输出的非线性映射，输出层对隐含层的输出实行线性加权组合，从而实现隐含层空间到输出层空间的线性变换。RBF网络与其它前馈网络的重要区别在于网络隐层神经元的非线性作用采用RBF激励函数作为隐层神经元的“基”构成隐含层空间。

七、进化计算

1. 遗传算法（GA）

遗传算法的基本操作包括复制、交叉和变异。复制是从一个旧种群中选择生命力强的个体（位申）产生新种群的过程，具有高适应度的位申更有可能在下一代中产生一个或多个子孙。复制操作可通过随机方法实现：产生0~1之间均匀分布的随机数，若某位申的复制概率为40%，则当随机数在0.40~1.0之间时该位申被复制，否则被淘汰。

遗传算法的特点在于：是对参数的编码进行操作而非对参数本身，可以借鉴生物学中染色体和基因等概念；同时使用多个搜索点的搜索信息，传统优化方法往往从解空间的单个初始点开始迭代搜索，单个搜索点提供的信息不多，搜索效率不高，容易陷入局部最优解。

2. 粒子群优化算法（PSO）

粒子群优化算法是通过模拟鸟类群体觅食行为而发展起来的一种基于群体协作的随机搜索算法，属于启发式全局优化算法，其基本思想是通过群体中个体之间的协作和信息共享来寻找最优解。

3. 差分进化算法（DE）

差分进化算法是模拟自然界生物种群以“优胜劣汰、适者生存”为原则的进化发展规律而形成的一种随机启发式搜索算法，于1995年由Rainer Storn和Kenneth Price提出，具有简单易用、稳健性好以及强大的全局搜索能力，已在多个领域取得成功。

八、物联网与智能网络

1. 物联网的定义

物联网是通过各种信息传感设备（如射频识别RFID、传感器、全球定位系统GPS、摄像机、激光扫描器等）和各种通讯手段（有线、无线、长距、短距），按约定的协议，实现人与人、人与物、物与物在任何时间、任何地点的连接，从而进行信息交换和通讯，以实现智能化识别、定位、跟踪、监控和管理的庞大网络系统。

温总理的定义强调物联网是在互联网基础上延伸和扩展的网络。欧盟的定义是将现有的互联的计算机网络扩展到互联的物品网络。国际电信联盟的定义指出物联网主要解决物品到物品（T2T）、人到物品（H2T）、人到人（HTH）之间的互连。

2. 智能网络与人工智能的应用

通信网络是信息流通最重要的基础设施，在通信网络应用人工智能技术拥有三大天然优势：数据量优势（通信网络中的各种网元、终端和业务系统每时每刻都在产生大量数据，如网元状态、链路流量、告警事件、信号质量、业务日志等，蕴含着大量有价值的信息）。

从网络层级来说，人工智能技术可以应用于链路网元层、网络管理控制层、业务运营编排层以及更上层的业务系统，利用数据回归、分类、推断、优化技术为网络服务。从流程上来说，可应用于网络的规划、设计、运维、优化以及客户服务等不同环节。从网络范围来讲，即可用于局部的链路通信性能优化增强，也可用于网元/小区级别的资源分配，到子网和网络级别的管理控制和协同。

3. 自智网络

自智网络是指能够提供零等待、零接触、零故障业务服务，并基于自服务、自组织、自保障的网络，这成为全球运营商共同期待的发展方向，也是5G时代运营商自身的重要核心竞争力之一。自智网络的核心是实现网络运营管理的线上化、自动化、智能化，通过B/O域融合的大数据分析、结合应用场景的人工智能、网络数字孪生技术构建以客户体验为中心的网络，实现配置管理自动化、业务编排自动化、运维诊断智能化等新型运营管理手段。

中国移动于2021年正式发布自智网络白皮书，提出了“三层四闭环”的内部实践目标框架，并设定了其网络运维自治水平在2025年达到L4的目标。中国联通提出了“四零四自”愿景目标（四零：零等待、零故障、零接触、零风险；四自：自规划、自配置、自修复、自优化）与“1+3+X”总体思路。

自智网络核心技术包括数据中台技术、通信人工智能、网络数字孪生三大支撑技术。数据是驱动包括人工智能、运营商数智化转型的最为重要的电子基础设施，高性能、低成本、面向智能化、高安全性的大型数据底座是自智网络赖以生存的基础。

4. 无线网络优化

无线网络优化是指按照一定的准则对通信网络的规划设计进行合理的调整，使网络运行更加可靠经济，网络服务质量更高，资源利用率更高。何时需要进行网络优化：建网或扩容工程之后；网络经过多次扩容和建设，系统遗留或积累了一些深层问题；网络存在质量问题，产生用户投诉并影响话费收入；面临对手有力的市场竞争；新功能、新技术应用后的进一步优化和完善；系统临近频带有其他系统对本系统产生干扰。

网络优化的常用方法包括用户投诉和CQT测试、信令跟踪分析法、话务统计分析法及路测分析法。其中分析OMC-R话务统计报告以及空中接口的跟踪分析是最常用的手段。

覆盖问题的表现主要有四种：覆盖空洞（连片站点中间出现完全没有5G信号的区域）、弱覆盖（有信号但信号强度不能保证稳定的KPI要求）、越区覆盖和导频污染。解决覆盖问题的手段按优先级包括：调整天线下倾角、调整天线方位角、调整RS的功率、升高或降低天线挂高、站点搬迁、新增站点或RRU。

九、密码学基础

1. 密码算法的分类

依据算法和密钥是否分开，可分为古典密码算法和现代密码算法。根据加密和解密是否使用相同的密钥，可分为对称密钥密码算法和非对称密钥密码算法（又称公开密钥密码算法）。根据每次操作的数据单元是否分块，可分为分组密码和序列密码。按编制原理可分为移位、代替和置换三种基本形式的组合。

2. 一次一密

一次一密要求通信双方首先安全交换一组密钥，每次交换的信息都选择一个不同的密钥。其要求包括：所有密钥必须随机产生；密钥的长度等于明文的长度；每个密钥只使用一次。

3. 移位密码

移位密码的典型代表是凯撒密码，密钥K=3时，明文abcdefghijklmnopqrstuvwxyz对应密文DEFGHIJKLMNOPQRSTUVWXYZABC，例如"caesar cipher"加密后为"FDHVDU FLSKHU"。

4. 对称加密算法

DES（数据加密标准） 属于常规密钥密码体制，是一种分组密码。在加密前先对整个明文进行分组，每个组长为64 bit，然后对每一个64 bit二进制数据进行加密处理，产生一组64 bit密文数据，最后将各组密文串接起来得出整个密文。使用的密钥为64 bit（实际密钥长度为56 bit，有8 bit用于奇偶校验）。DES的保密性仅取决于对密钥的保密，而算法是公开的，至今仍未能找到比穷举搜索密钥更有效的方法。

IDEA算法 的分组长度为64 bits，密钥长度为128 bits，同一算法既可以加密也可用于解密，只需要简单的混合运算（异或、模2^16加、模(2^16+1)乘），软件实现比DES快两倍。

对称加密算法的加密模式 包括：电子代码本模式（ECB）、密码块链模式（CBC）、密码反馈模式（CFB）、输出反馈模式（OFB）。

5. 公开密钥密码体制

公开密钥密码体制的基本思想是：每个用户产生一对密钥PK和SK，加密密钥PK公开，解密密钥SK保密，加密算法E和解密算法D也都是公开的。私有密钥SK由公开密钥PK决定，但不能根据PK计算出SK。任何加密方法的安全性取决于密钥的长度以及攻破密文所需的计算量，在这方面公开密钥密码体制并不具有比传统加密体制更加优越之处。

使用公开密钥进行加密通信时，每个通信实体有一对密钥（公钥和私钥），公钥用于加密和验证签名，私钥用于解密和签名。在签名应用中，发送方用私钥加密（签名），接收方收到密文后用发送方的公钥解密（验证）。

6. RSA公开密钥算法

RSA所根据的原理是：根据数论，寻求两个大素数比较简单，而将它们的乘积分解开则极其困难。每个用户有两个密钥：加密密钥PK={e,n}和解密密钥SK={d,n}，用户把加密密钥公开，而对解密密钥中的d保密。n为两个大素数p和q之积（p和q一般为100位以上的十进制数），e和d满足一定的关系，当敌手已知e和n时并不能求出d。

密钥产生举例：取素数p=7，q=17，则n=p×q=119，Φ(n)=(7-1)×(17-1)=96；选择e=5，满足gcd(e,Φ(n))=1且1<e<Φ(n)；选择d满足5d=k×96+1，令k=4，求得d=77。所以公钥KU={5,119}，私钥KR={77,119}。

7. 单向散列函数与消息认证

散列函数是一种算法，它将任意长度的报文M变换成固定长度的输出，算法的输出称为散列码或者报文摘要。报文摘要唯一地对应原始报文，散列函数能用来检测报文的完整性，保证报文没有被改变和破坏。

认证技术包括对用户身份的认证和对消息的认证两种。实体认证即验证信息发送者是真的而不是冒充的，包括信源、信宿等的认证和识别；消息认证验证信息的完整性，即验证数据在传输或存储过程中未被篡改、重放或延迟。

8. 数字签名

数字签名具有以下特点：可验证（接受者可以核实发送者对报文的签名）；防抵赖（发送者事后不能抵赖对报文的签名）；防假冒（攻击者不能冒充发送者向收方发送文件）；防篡改（接收者不能对收到的文件进行篡改）；防伪造（接收者不能伪造对报文的签名）。

仲裁数字签名引入仲裁者，所有从发送方X到接收方Y的签名消息首先送到仲裁者A，A将消息及其签名进行一系列测试以检查其来源和内容，然后A将消息加上日期并与已被仲裁者验证通过的指示一起发给Y。仲裁者在这一类签名模式中扮演敏感和关键的角色，所有参与者必须极大地相信这一仲裁机制工作正常。

二、防火墙原理与技术

1. 防火墙核心原则

  • ✅ 所有进出通信必须经过防火墙
  • ✅ 所有通过流量必须过滤或授权
  • ✅ 防火墙本身不可被侵入

2. 附加功能（三大件）

  • 流量控制：按用户限流，合理分配带宽
  • NAT：修改IP/端口，节省公网IP、隐藏内网
  • VPN：封装加密，通过公网安全传输私有数据

3. 防火墙的局限性（必考）

  • ❌ 无法防范绕过防火墙的攻击（如内部用户私自拨号）
  • ❌ 无法防止病毒文件传输（需主机反病毒软件）
  • ❌ 无法防止数据驱动式攻击（看似无害数据被执行后攻击）
  • ❌ 无法防范恶意内部人员（不经过防火墙）

1. 定义

动态监控、预防或抵御入侵行为的安全机制，监控网络/系统状态和行为，检测越权使用和入侵企图。

3. 优缺点速记

优点：提高信息安全构造的其他部分的完整性，提高系统的监控能力，从入口点到出口点跟踪用户的活动，识别和汇报数据文件的变化，侦测系统配置错误并纠正它们，识别特殊攻击类型并向管理人员发出警报进行防御。

缺点：不能弥补差的认证机制，如果没有人的干预不能管理攻击调查，不能知道安全策略的内容，不能弥补网络协议上的弱点，不能弥补系统提供质量或完整性的问题，不能分析一个堵塞的网络。

4. 散列算法（完整性检测）

在入侵检测系统（IDS）中，散列算法主要用于验证数据或文件的完整性，而非直接检测网络流量中的实时攻击。其核心原理是利用散列函数的单向性和敏感性。

在系统干净时，对关键文件（如系统二进制文件、配置文件）计算散列值（如MD5、SHA-1、SHA-256），并将该值安全存储为基准。

然后进行周期性重计算,并比对检测，将新计算的散列值与保存的基准值比对，从而判断是否遭受入侵。

  • 常用算法：MD5、CRC16/32、SHA、MD4、MD2、Snefru、Haval
  • 三大特性：
    1. 易用性：任意长度m → 固定长度h很容易
    2. 单向性：由h反推m极其困难,H(m) = h
    3. 无碰撞性：很难找到另一个m',能使H(m')=H(m)

5. 决策树（ID3算法）

  • 提出者：Quinlan，1979

  • 核心：以信息熵下降速度选择属性

  • 用“问问题”的方式做决策，每个问题把数据分成更纯的子集

    ​ $\text{信息增益}(D, A) = \text{熵}(D) - \sum_{v=1}^{V} \frac{|D_v|}{|D|} \cdot \text{熵}(D_v)$

  • 步骤：

    1. 根结点包含所有实例
    2. 若叶结点内同一分类则停止
    3. 否则选信息熵最小的属性a_best
    4. 按a_best的值划分分枝
    5. 重复步骤2

6. 集成学习（元算法）

  • 思想：多个分类器组合，效果优于单个 ---- 组合多个弱学习器，得到强学习器

  • Boosting（以 AdaBoost 为代表）：

    • 串行训练，每次关注上一轮预测错误的样本
    • 精度高
    • 框架式，可使用各种子分类器
    • 弱分类器构造简单
    • 无需特征筛选
    • 不易过拟合

  • Bagging（自举汇聚法）：

    • 并行训练，在每个训练集上独立训练一个基模型，投票/平均

    • 最后是投票还是平均，看问题类型

      问题类型输出类型聚合方式例子
      分类离散的类别（猫/狗、好/坏）投票5 个模型：3 个说“猫”，2 个说“狗” → 最终“猫”
      回归连续的数值（价格、温度）平均5 个模型预测：$280, 310, 295, 305, 290$ → 平均数296

    • 有放回抽样生成s个新数据集

    • 每个数据集训练一个分类器

    • 新数据由s个分类器投票决定

  • 打球=是：序号 3、4 → 2条
    ($ P(是) = \frac{2}{5} = 0.4$ )
  • 打球=否：序号 1、2、5 → 3条
    ( $P(否) = \frac{3}{5} = 0.6$)

✅ 这一步的意义：统计训练数据中各类别的占比，作为“先验信念”。

第2步：计算条件概率（似然）

给定“打球=是”的情况下：

  • P(天气=阴 | 是) = 1/2 = 0.5
  • P(湿度=高 | 是) = 1/2 = 0.5
  • P(有风=否 | 是) = 2/2 = 1.0

给定“打球=否”的情况下：

  • P(天气=阴 | 否) = 0/3 = 0
  • P(湿度=高 | 否) = 2/3 ≈ 0.667
  • P(有风=否 | 否) = 1/3 ≈ 0.333

✅ 这一步的意义：从训练集中统计在每一类下，各特征取特定值的比例。

第3步：计算后验概率（未归一化）

$ P(是|X) \propto P(是) \times P(阴|是) \times P(高|是) \times P(否|是) = 0.4 \times 0.5 \times 0.5 \times 1.0 = 0.1 $

$ P(否|X) \propto 0.6 \times 0 \times 0.667 \times 0.333 = 0 $

✅ 这一步的意义：利用贝叶斯公式，结合先验和似然，计算每种类别的可能性。

第4步：比较并分类

因为 (P(否|X)=0)（实际可用拉普拉斯平滑避免零概率），
所以判定为 “打球=是”。

✅ 最终结论：该样本被分到“是”类。

二、决策树（ID3）计算题

对于数据集 ( D_v )，其信息熵 (\text{熵}(D_v)) 的公式为：

$ \boxed{\text{Ent}(D_v) = -\sum_{k=1}^{|y|} p_k \log_2 p_k} $

其中：

  • $ |y| $：类别总数。
  • $ p_k $：在 ( D_v ) 中，第 ( k ) 类样本所占的比例（即 ( p_k = \frac{|C_k|}{|D_v|} )，( |C_k| ) 为第 ( k ) 类的样本数，( |D_v| ) 为 ( D_v ) 总样本数）。
  • 通常约定 $ 0 \log_2 0 = 0$。

简记形式（二分类时常用）：

$ \text{Ent}(D_v) = -p_+ \log_2 p_+ - p_- \log_2 p_- $

  • $ p_+$：正例子所占比例。
  • $p_- $：负例子所占比例。

信息增益 = 划分前的熵 - 划分后的加权平均熵

$\text{信息增益}(D, A) = \text{熵}(D) - \sum_{v=1}^{V} \frac{|D_v|}{|D|} \cdot \text{熵}(D_v)$

题目： 根据以下天气数据，判断条件为：天气=阴天、湿度=高、有风=否 时，是否适合打球？

序号天气湿度有风打球？
1晴高否否
2晴高是否
3阴高否是
4雨正常否是
5雨正常是否

使用相同数据，选择第一个分裂特征。

第1步：计算整个数据集的信息熵

此处，打球=是：2条，占比2/5，打球=否：3条 占比3/5 $ H(S) = -\frac{2}{5}\log_2\frac{2}{5} - \frac{3}{5}\log_2\frac{3}{5} = -0.4 \times (-1.322) - 0.6 \times (-0.737) \approx 0.971 $

✅ 这一步的意义：衡量当前数据集的“混乱程度”。

第2步：计算每个特征的信息增益

特征：天气

  • 晴 → 打球：0/2=0，不打球：2/2=1 → 熵=0
  • 阴 → 打球：1/1=1，不打球：0 → 熵=0
  • 雨 → 打球：1/2，不打球：1/2 → 熵=1

加权平均熵： $ \frac{2}{5} \times 0 + \frac{1}{5} \times 0 + \frac{2}{5} \times 1 = 0.4 $

信息增益： $ Gain(天气) = 0.971 - 0.4 = 0.571 $

特征：湿度

  • 高 → 打球：1/3，不打球：2/3
    熵 = -1/3·log₂(1/3) - 2/3·log₂(2/3) ≈ 0.918
  • 正常 → 打球：1/2，不打球：1/2 → 熵=1

加权熵： $ \frac{3}{5} \times 0.918 + \frac{2}{5} \times 1 = 0.951 $ $ Gain(湿度) = 0.971 - 0.951 = 0.020 $

特征：有风

  • 是 → 打球：0/2=0，不打球：1 → 熵=0
  • 否 → 打球：2/3，不打球：1/3
    熵 = -2/3·log₂(2/3) - 1/3·log₂(1/3) ≈ 0.918

加权熵： $ \frac{2}{5} \times 0 + \frac{3}{5} \times 0.918 = 0.551 $ $ Gain(有风) = 0.971 - 0.551 = 0.420 $

✅ 这一步的意义：信息增益越大，说明该特征能更有效地降低不确定性。

第3步：选择分裂特征

比较信息增益：

  • 天气：0.571
  • 湿度：0.020
  • 有风：0.420

选 天气 作为根节点。

✅ 这一步的意义：选择当前最优特征对数据集进行划分。

第4步（可选）：继续分裂

  • 若天气=晴 → 全是“不打球” → 叶节点
  • 若天气=阴 → 全是“打球” → 叶节点
  • 若天气=雨 → 混合（1打球，1不打球）→ 在子集上重复上述过程选择下一特征

✅ 最终得到一棵决策树。
