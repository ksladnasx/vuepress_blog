前端 性能优化指南

前端性能优化是个大话题，我直接给你最实用、最高频的优化手段，分通用和框架特有两部分。

一、通用优化（Vue 和 React 都适用）

1.1 虚拟列表——渲染大量数据

问题：渲染 10000 条列表，页面卡死

解决：只渲染可视区域的内容

框架推荐库代码示例
Vuevue-virtual-scroller<RecycleScroller :items="list" :item-size="50">
Reactreact-window<FixedSizeList height={600} itemCount={1000} itemSize={35}>

1.2 路由懒加载——首屏加载更快

Vue：

// 原来：import About from './views/About.vue'
// 优化后：
const About = () => import('./views/About.vue')

const routes = [
  { path: '/about', component: About }  // 访问时才加载
]

React：

// React 18+
const About = lazy(() => import('./views/About'))

function App() {
  return (
    <Suspense fallback={<div>加载中...</div>}>
      <About />
    </Suspense>
  )
}

1.3 图片优化

<!-- 懒加载：滚动到才加载 -->
<img v-lazy="imageUrl" />

<!-- 使用 WebP 格式（体积小 30%） -->
<picture>
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg">
</picture>

<!-- 响应式图片 -->
<img srcset="small.jpg 480w, large.jpg 1080w" sizes="(max-width: 600px) 480px, 1080px">

1.4 防抖和节流

// 搜索输入防抖（停止输入 500ms 后才请求）
import { debounce } from 'lodash'
const search = debounce(async (keyword) => {
  await fetchAPI(keyword)
}, 500)

// 滚动节流（每 200ms 最多执行一次）
import { throttle } from 'lodash'
const handleScroll = throttle(() => {
  console.log('滚动位置', window.scrollY)
}, 200)

1.5 CDN 加速 + Gzip

配置（以 Vite 为例）：

// vite.config.js
export default {
  build: {
    rollupOptions: {
      external: ['vue', 'vue-router'],  // 排除打包，改用 CDN
    }
  }
}

nginx 开启 Gzip：

gzip on;
gzip_types text/css application/javascript application/json;
gzip_min_length 1000;

二、Vue 特有优化

2.1 v-for 必须加 key（这个你肯定知道）

<!-- ✅ 正确 -->
<div v-for="item in list" :key="item.id">{{ item.name }}</div>

<!-- ❌ 错误：用 index 会导致性能问题和状态错乱 -->
<div v-for="(item, index) in list" :key="index">{{ item.name }}</div>

2.2 使用 v-show 而不是 v-if（频繁切换的场景）

<!-- v-if：频繁切换会不断销毁重建，开销大 -->
<div v-if="isVisible">内容</div>

<!-- v-show：只是切换 display，适合频繁切换 -->
<div v-show="isVisible">内容</div>

2.3 计算属性缓存 vs 方法调用

<script setup>
import { computed } from 'vue'

// ✅ 计算属性：依赖不变就不重新计算（有缓存）
const expensiveResult = computed(() => {
  return heavyCalculation(list.value)
})

// ❌ 方法：每次渲染都会重新计算
const expensiveMethod = () => {
  return heavyCalculation(list.value)
}
</script>

<template>
  <!-- 多次使用，计算属性只算一次，方法算多次 -->
  <div>{{ expensiveResult }}</div>
  <div>{{ expensiveResult }}</div>
</template>

2.4 组件异步加载

// 原来：import HeavyComponent from './HeavyComponent.vue'

// 优化后：按需加载
const HeavyComponent = defineAsyncComponent(() => 
  import('./HeavyComponent.vue')
)

2.5 使用 shallowRef 优化大对象

import { shallowRef } from 'vue'

// 普通 ref：对象内部任何属性变化都会触发更新
const data = ref({ name: 'John', address: {...}, orders: [...] })

// shallowRef：只监听 .value 的变化，内部属性变化不触发更新（性能好）
const bigData = shallowRef({ users: [...10000条数据] })
// 修改整个数据才触发更新
bigData.value = { users: [...] }

三、React 特有优化

3.1 memo 避免子组件无效渲染

import { memo } from 'react'

// 父组件状态变化，Child 不会重新渲染（props 没变）
const Child = memo(({ name }) => {
  console.log('Child 渲染了')
  return <div>{name}</div>
})

function Parent() {
  const [count, setCount] = useState(0)
  return (
    <div>
      <button onClick={() => setCount(count + 1)}>点击 {count}</button>
      <Child name="John" />  {/* 这个不会重新渲染 */}
    </div>
  )
}

3.2 useCallback + useMemo 缓存

import { useCallback, useMemo, useState } from 'react'

function SearchPage() {
  const [keyword, setKeyword] = useState('')
  const [filter, setFilter] = useState('all')
  
  // 缓存计算结果（类似 Vue 的 computed）
  const filteredList = useMemo(() => {
    console.log('重新计算过滤列表')
    return expensiveFilter(allData, keyword, filter)
  }, [keyword, filter])  // 只有依赖变化才重新计算
  
  // 缓存函数引用（防止子组件不必要的渲染）
  const handleSearch = useCallback((value) => {
    setKeyword(value)
    fetchResults(value)
  }, [])  // 空依赖 = 永远不变
  
  return <SearchInput onSearch={handleSearch} />
}

3.3 使用 React.lazy + Suspense

import { lazy, Suspense } from 'react'

// 路由懒加载
const Dashboard = lazy(() => import('./pages/Dashboard'))
const Settings = lazy(() => import('./pages/Settings'))

function App() {
  return (
    <Suspense fallback={<div>加载中...</div>}>
      <Routes>
        <Route path="/dashboard" element={<Dashboard />} />
        <Route path="/settings" element={<Settings />} />
      </Routes>
    </Suspense>
  )
}

3.4 避免内联对象/函数

// ❌ 每次渲染都创建新对象，导致子组件重新渲染
function Parent() {
  return <Child style={{ color: 'red' }} onClick={() => console.log('click')} />
}

// ✅ 提取到外部
const childStyle = { color: 'red' }
const handleClick = () => console.log('click')

function Parent() {
  return <Child style={childStyle} onClick={handleClick} />
}

3.5 使用 Fragment 减少 DOM 层级

// ❌ 多余的 div
function Component() {
  return (
    <div>
      <Header />
      <Content />
      <Footer />
    </div>
  )
}

// ✅ 使用 Fragment
function Component() {
  return (
    <>
      <Header />
      <Content />
      <Footer />
    </>
  )
}

五、实战优先级排序

如果你是第一次优化项目，按这个顺序做收益最大：

优先级优化手段预期收益
🔴 P0路由懒加载首屏加载时间 ↓ 30-50%
🔴 P0图片懒加载 + WebP首屏加载时间 ↓ 20-40%
🟠 P1v-for 加 key / React memo列表渲染性能 ↑ 明显
🟠 P1开启 Gzip 压缩资源体积 ↓ 60-80%
🟡 P2虚拟列表（超长列表）页面卡顿 → 流畅
🟡 P2防抖节流减少无效请求
🟢 P3useMemo/计算属性减少重复计算

六、快速检查清单

□ 路由是否做了懒加载？
□ 图片是否用了懒加载 + WebP 格式？
□ 列表渲染是否加了正确的 key？
□ 搜索/滚动事件是否做了防抖/节流？
□ 有没有不必要的组件重复渲染？（用 DevTools 看）
□ 生产环境是否开启了 Gzip？
□ 第三方库是否按需引入？（如 lodash、element-plus）
□ 有没有使用 CDN 加载公共库？（vue/react/axios）

总结：先做路由懒加载和图片优化（收益最大），再用DevTools 找渲染瓶颈针对性优化（别过早优化），最后用Lighthouse 跑分验证效果。
