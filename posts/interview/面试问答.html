面试问答模拟

HTML部分

面试官： 先从 HTML 开始吧，你觉得语义化标签为什么重要？

面试者： 语义化标签的核心是“标签本身具有含义”，比如 header、nav、main、article、section、footer 等。这样做有几个好处：第一是代码结构更清晰，团队协作更容易；第二是 SEO 更友好，搜索引擎更容易理解页面内容；第三是无障碍访问更好，屏幕阅读器能更准确识别页面结构。实际开发里我会尽量避免大量 div 嵌套。

面试官： script 标签中的 defer 和 async 有什么区别？

面试者： 两者都是为了解决 JS 阻塞 HTML 解析的问题。 async 会异步下载脚本，下载完成后立即执行，执行顺序无法保证，适合统计脚本、广告脚本。 defer 也是异步下载，但会等 HTML 解析完成后按顺序执行，更适合业务脚本。 如果多个脚本之间有依赖关系，我会优先使用 defer。

面试官： localStorage、sessionStorage 和 cookie 的区别？

面试者： cookie 会随着 HTTP 请求自动发送给服务器，大小一般只有 4KB 左右，常用于登录态。 localStorage 是永久存储，除非手动清除，否则不会消失。 sessionStorage 生命周期只在当前会话页。 localStorage 和 sessionStorage 都不会自动携带到服务端，容量一般比 cookie 大很多。

面试官： iframe 有哪些缺点？

面试者： iframe 会增加页面加载开销，因为它本质是嵌套了另一个浏览器上下文；SEO 不友好；移动端兼容体验一般；还容易产生跨域问题。另外多个 iframe 会占用更多内存。现在除了第三方页面嵌入，一般不会大规模使用。

CSS 部分

面试官： 说一下 CSS 选择器优先级。

面试者： !important 最高。 然后是内联样式。 接着是 id 选择器。 再是 class、属性选择器、伪类。 最后是标签选择器和伪元素。 如果优先级相同，则后面的样式覆盖前面的。

面试官： BFC 是什么？

面试者： BFC 即块级格式化上下文，它是一个独立的渲染区域，内部元素不会影响外部。常见触发方式有：

  • overflow: hidden
  • display: flex
  • position: absolute/fixed
  • float 不为 none

BFC 常用于解决高度塌陷、margin 重叠等问题。

面试官： 什么是高度塌陷？怎么解决？

面试者： 当父元素内部子元素全部浮动后，子元素脱离文档流，父元素无法感知高度，就会出现高度塌陷。 解决方案有：

  1. 父元素触发 BFC
  2. clear: both
  3. 使用伪元素 clearfix
  4. 直接使用 flex 布局避免 float

现在实际开发中 float 已经比较少了，更多会用 flex。

面试官： position 有哪些值？区别是什么？

面试者： static：默认定位。 relative：相对自身原位置定位，不脱离文档流。 absolute：相对最近的非 static 父元素定位，脱离文档流。 fixed：相对视口定位。 sticky：粘性定位，结合 relative 和 fixed 特性。

面试官： flex: 1 是什么意思？

面试者： flex:1 实际上是：

flex-grow: 1;
flex-shrink: 1;
flex-basis: 0%;

表示元素会平分剩余空间，并且在空间不足时允许缩小。

面试官： rem、em、vw 的区别？

面试者： em 相对于父元素字体大小。 rem 相对于根元素 html 的字体大小。 vw 相对于视口宽度。

移动端响应式开发里我比较常用 rem 和 vw。

面试官： CSS 动画有哪些实现方式？

面试者： 主要有 transition 和 animation。 transition 更适合简单状态切换。 animation 配合 keyframes 更适合复杂动画。

性能优化方面我会优先使用 transform 和 opacity，因为它们通常只触发合成层，不容易引发重排。

JavaScript 部分

面试官： 说一下原型和原型链。

面试者： JavaScript 是基于原型继承的。 每个对象都有一个隐式原型 proto，指向其构造函数的 prototype。 查找属性时，如果当前对象没有，就会沿着原型链向上查找，直到 Object.prototype。 这就是原型链。

面试官： new 一个对象时发生了什么？

面试者： new 的过程主要有四步：

  1. 创建空对象
  2. 将对象原型指向构造函数 prototype
  3. 执行构造函数并绑定 this
  4. 返回对象

等价于手动实现：

function myNew(fn, ...args) {
  const obj = Object.create(fn.prototype)
  const result = fn.apply(obj, args)
  return result instanceof Object ? result : obj
}

面试官： call、apply、bind 区别？

面试者： 三者本质都是改变 this 指向。 call 参数逐个传。 apply 参数是数组。 bind 不会立即执行，而是返回新函数。

面试官： this 指向有哪些规则？

面试者： 默认绑定：普通函数指向 window。 隐式绑定：谁调用指向谁。 显式绑定：call/apply/bind 指定。 new 绑定：指向实例对象。 箭头函数没有自己的 this，会继承外层作用域。

面试官： 箭头函数为什么不能当构造函数？

面试者： 因为箭头函数没有 prototype，也没有自己的 this，不能使用 new 调用。

面试官： 深拷贝和浅拷贝区别？

面试者： 浅拷贝只复制第一层引用。 深拷贝会递归复制所有层级。

常见浅拷贝：

Object.assign()
展开运算符

深拷贝：

structuredClone()
JSON.parse(JSON.stringify())

但 JSON 方案会丢失 undefined、Symbol、函数等。

面试官： Promise 有几种状态？

面试者： pending、fulfilled、rejected。 状态一旦改变就不可逆。

面试官： Promise.then 为什么能链式调用？

面试者： 因为 then 本身返回的是一个新的 Promise。 如果返回普通值，会包装成 resolve。 如果返回 Promise，则会等待它完成。

面试官： async/await 本质是什么？

面试者： 本质是 Promise 的语法糖。 async 函数默认返回 Promise。 await 会暂停函数执行，等待 Promise resolve 后继续。

面试官： 什么是防抖和节流？

面试者： 防抖：频繁触发后只执行最后一次，比如搜索框输入。 节流：固定时间内只执行一次，比如滚动事件。

面试官： 手写一个防抖。

function debounce(fn, delay) {
  let timer = null

  return function (...args) {
    clearTimeout(timer)

    timer = setTimeout(() => {
      fn.apply(this, args)
    }, delay)
  }
}

面试官： 说一下事件冒泡和事件捕获。

面试者： 事件流分三阶段：

  1. 捕获阶段
  2. 目标阶段
  3. 冒泡阶段

默认 addEventListener 是冒泡。 第三个参数传 true 可以改成捕获。

event.stopPropagation() 可以阻止冒泡。

面试官： 什么是事件委托？

面试者： 利用事件冒泡，把子元素事件统一绑定到父元素上。 优点是减少事件监听数量，提高性能，并且适合动态元素。

面试官： let、const、var 区别？

面试者： var 存在变量提升，没有块级作用域。 let 有块级作用域，但不能重复声明。 const 也有块级作用域，并且声明后不能修改引用。

另外 let 和 const 存在暂时性死区。

面试官： 为什么 0.1 + 0.2 !== 0.3？

面试者： 因为 JavaScript 使用 IEEE754 双精度浮点数存储，二进制无法精确表示某些小数，导致精度误差。

通常会这样处理：

(0.1 + 0.2).toFixed(2)

或者：

Math.abs(0.1 + 0.2 - 0.3) < Number.EPSILON

面试官： 说一下浏览器缓存。

面试者： 强缓存和协商缓存。 强缓存通过 Expires 和 Cache-Control 控制，不请求服务器。 协商缓存通过 ETag 和 Last-Modified 与服务器比对。

实际项目里静态资源一般会配合 hash 做长期缓存。
