自然，利用shift可以做队列（先进先出），利用pop可以实现栈(先进后出),同时还有slice 和 splice ：

这俩名字长得像，但用法和功能完全不一样。最大的区别就是：slice 是不改变原数组的，用来截取一部分；splice 是直接操作原数组的，用来增删改。

slice 就是切片的意思。用法是 arr.slice(start, end)，从 start 截取到 end（不包括 end），返回一个新数组，原数组纹丝不动。如果没传参数，就复制整个数组，这也是浅拷贝的一种常用方式。比如 arr.slice() 就能得到一个新数组。另外 start 和 end 支持负数，-1 表示最后一个元素。

splice 就是拼接/手术刀的意思。用法是 arr.splice(start, deleteCount, ...items)，它会从 start 位置开始，删除 deleteCount 个元素，然后再把后面传进来的 items 插入进去。返回值是被删掉的元素组成的新数组。最重要的是，它直接改变了原数组。比如 arr.splice(1, 2) 就是从索引 1 开始删 2 个元素；

如果 deleteCount 是 0，就只插入不删除，比如 arr.splice(1, 0, '新元素')；如果只传 start，就会删掉从 start 到末尾的所有元素。

所以总结一下：

  • slice 用来取数据，不改变原数组
  • splice 用来改数据，会改变原数组

面试的时候经常问，记住“不改变用 slice，增删改用 splice”就行。另外字符串也有 slice，但字符串没有 splice。

接雨水问题

给定 n 个非负整数表示每个宽度为 1 的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。

核心是要理解，对于每一个位置（数组中每一个数），它能接的雨水量取决于：

min(左边最大高度, 右边最大高度) - 当前高度

双指针法：

  1. 用两个指针从两端向中间移动
  2. 维护左右两边的最大高度
  3. 每次处理较矮的那一边（因为矮边决定了水位上限）
def trap_rain_water(height):
    """
    计算接雨水的总量
    思路：双指针法，时间复杂度 O(n)
    """
    if not height or len(height) < 3:
        return 0
    
    left, right = 0, len(height) - 1
    left_max, right_max = 0, 0
    water = 0
    
    while left < right:
        # 哪边矮，就先处理哪边
        if height[left] < height[right]:
            # 更新左边最大值
            if height[left] >= left_max:
                left_max = height[left]
            else:
                # 当前位置能接的水量 = 左边最大高度 - 当前高度
                water += left_max - height[left]
            left += 1
        else:
            # 更新右边最大值
            if height[right] >= right_max:
                right_max = height[right]
            else:
                # 当前位置能接的水量 = 右边最大高度 - 当前高度
                water += right_max - height[right]
            right -= 1
    
    return water

if __name__ == '__main__':
    n = list(map(int, input().split(",")))
    print(trap_rain_water(n))

动态规划法（更直观）:

def trap_rain_water_dp(height):
    """
    动态规划法：计算每个位置的左右最大高度
    """
    if len(height) < 3:
        return 0
    
    n = len(height)
    
    # 1. 计算每个位置左边的最大高度
    left_max = [0] * n
    left_max[0] = height[0]
    for i in range(1, n):
        left_max[i] = max(left_max[i-1], height[i])
    
    # 2. 计算每个位置右边的最大高度
    right_max = [0] * n
    right_max[n-1] = height[n-1]
    for i in range(n-2, -1, -1):
        right_max[i] = max(right_max[i+1], height[i])
    
    # 3. 计算总水量
    water = 0
    for i in range(n):
        # 当前位置能接的水量 = min(左边最大, 右边最大) - 当前高度
        water += min(left_max[i], right_max[i]) - height[i]
    
    return water

    n = list(map(int, input().split(",")))
    print(trap_rain_water_dp(n))

N皇后问题

按照国际象棋的规则，皇后可以攻击与之处在同一行或同一列或同一斜线上的棋子。

n 皇后问题 研究的是如何将 n 个皇后放置在 n×n 的棋盘上，并且使皇后彼此之间不能相互攻击。

给你一个整数 n ，返回所有不同的 n 皇后问题 的解决方案。

每一种解法包含一个不同的 n 皇后问题 的棋子放置方案，该方案中 'Q' 和 '.' 分别代表了皇后和空位。

行和列好避免，现在主要问题是同一个斜线，实际上要理解的是，如果是主对角线：左上到右下，特点是行数减去列数是常数，即row - col = 常数。则副对角线：右上到左下，特点：row + col = 常数。

那要做很简单，首先肯定不在同一行，我们可以通过queens 列表，记录每行皇后所在的列位置，例如: queens = [1, 3, 0, 2] 表示：第0行皇后在第1列，第1行皇后在第3列。再用xy_diff: 集合记录已占用的主对角线的常数，用 xy_sum: 集合，记录已占用的副对角线，里面存的都是常数，那在判断的时候尝试在当前行的每一列放置皇后，如果列号q不在queens 列表中，且行号与列号的差值和和值不在集合中，说明条件满足，进行皇后放置（递归调用函数）

"""
N皇后问题 - 回溯法
如果 q 已经在 queens 中，说明这一列已经有皇后了
# p: 当前行号（从0开始）
# q: 当前列号（从0开始）
# xy_diff: 集合，记录已占用的主对角线
# xy_sum: 集合，记录已占用的副对角线
"""
def solveNQueens(n):

    def dfs(queens, xy_diff, xy_sum):
        """
        queens: 每行皇后所在的列位置
        xy_diff: 主对角线 (row - col) 集合
        xy_sum:  副对角线 (row + col) 集合
        """
        p = len(queens)  # 当前行数
        if p == n:  # 找到一种解决方案
            result.append(queens[:])
            return

        for q in range(n):  # 尝试在当前行的每一列放置皇后
            # 检查是否冲突：同列、主对角线、副对角线
            if q not in queens and p - q not in xy_diff and p + q not in xy_sum:
                # 放置皇后，继续下一行
                dfs(queens + [q], xy_diff | {p - q}, xy_sum | {p + q})

    result = []
    dfs([], set(), set())

    # 转换为棋盘表示
    boards = []
    for solution in result:
        board = []
        for col in solution:
            row_str = '.' * col + 'Q' + '.' * (n - col - 1)
            board.append(row_str)
        boards.append(board)

    return boards

寻找链表交点

给你两个单链表的头节点 headA 和 headB ，请你找出并返回两个单链表相交的起始节点。如果两个链表不存在相交节点，返回 null 。

若两个链表有交点，记录交点后走的公共部分长度为c，交点前的部分的长度为a和b，显然a+c+b = b+c +a，即将A链表头节点遍历一遍之后再从B的头节点开始遍历到交点处所走路径等于B链表遍历一遍之后再从A的头节点开始遍历到交点处所走路径。

var getIntersectionNode = function(headA, headB) {
    // 只要有一个是空，肯定不相交
    if (headA === null || headB === null) return null;
    
    let pA = headA;
    let pB = headB;
    
    // 当两个指针相遇时退出循环
    while (pA !== pB) {
        // 如果pA走到头了，就切到headB继续走；否则走下一步
        pA = (pA === null ? headB : pA.next); 
        // pB同理，走到头就切到headA
        pB = (pB === null ? headA : pB.next);
    }
    // 如果相交，pA和pB就是交点；如果不相交，最终都会是null
    return pA;
};

二叉树节点找最近共同祖先

给定二叉树根节点，寻找给定的两个节点的最近共同祖先，两节点不相同。

这个题我之前刷过，思路其实就是用递归去遍历，找到 p 和 q 的位置，然后往上返回，本质上就是二叉树的搜索。写法不算复杂，我直接写出来吧：

var lowestCommonAncestor = function(root, p, q) {
    // 如果当前节点是空，或者就是 p 或 q 本身，直接返回当前节点
    if (root === null || root === p || root === q) {
        return root;
    }
    
    // 去左子树里找 p 和 q
    const left = lowestCommonAncestor(root.left, p, q);
    // 去右子树里找 p 和 q
    const right = lowestCommonAncestor(root.right, p, q);
    
    // 如果左子树找到了、右子树也找到了，那当前节点就是最近公共祖先
    if (left !== null && right !== null) {
        return root;
    }
    
    // 否则哪边不为空就返回哪边
    return left !== null ? left : right;
};

我解释一下这个过程。递归会一直往下走，直到找到 p 或 q 或者叶子节点。当某个节点左子树和右子树分别找到了 p 和 q，说明这个节点就是它们的最近公共祖先，因为 p 和 q 分别挂在它的左右两边。如果只有一边找到了，那就向上返回找到的那个节点，说明 p 和 q 都在那一边，公共祖先还在上面。

这个算法的复杂度是 O(n)，每个节点最多访问一次。

/**
 * Definition for singly-linked list.
 */
//定义链表类
class ListNode {
  constructor(val, next) {
    this.val = val === undefined ? 0 : val;
    this.next = next === undefined ? null : next;
  }
}
/**
 * @param {ListNode} head
 * @return {boolean}
 */
var isPalindrome = function (head) {
  let arr = [];
  let current = head;
  while (current) {
    arr.push(current.val);
    current = current.next;
  }

  for (let i = 0; i < arr.length / 2; i++) {
    if (arr[i] !== arr[arr.length - 1 - i]) {
      return false;
    }
  }

  return true;
};

每日温度

给定一个整数数组 temperatures ，表示每天的温度，返回一个数组 answer ，其中 answer[i] 是指对于第 i 天，下一个更高温度出现在几天后。如果气温在这之后都不会升高，请在该位置用 0 来代替。

这个题叫“每日温度”，典型的单调栈应用。我写一下代码，然后解释思路：

var dailyTemperatures = function(temperatures) {
    const n = temperatures.length;
    const answer = new Array(n).fill(0);
    const stack = []; // 存的是索引，不是温度值
    
    for (let i = 0; i < n; i++) {
        // 当栈不为空，且当前温度比栈顶索引对应的温度高时，说明找到了下一个更高温度
        while (stack.length > 0 && temperatures[i] > temperatures[stack[stack.length - 1]]) {
            const prevIndex = stack.pop();
            answer[prevIndex] = i - prevIndex;
        }
        // 当前索引入栈
        stack.push(i);
    }
    
    return answer;
};

思路是这样的：我们维护一个单调递减栈，栈里存的是还没找到下一个更高温度的那些天的索引。遍历每一天，如果当前温度比栈顶那天的温度高，就说明栈顶那天遇到了第一个比它热的日子，计算天数差填进结果，然后把栈顶弹出来。重复这个过程，直到栈为空或者当前温度不比栈顶高。最后把当前天的索引进栈。

等遍历完，还在栈里的那些索引说明后面没有比它们温度更高的日子了，结果数组本来就是 0，所以不用额外处理。

这个算法时间复杂度 O(n)，每个元素入栈出栈各一次；空间复杂度 O(n) 最坏情况就是温度一直递减，栈里会存 n 个元素。

为什么时间复杂度是 O(n)而不是O(n²) ？

关键在于：每个元素最多被入栈一次、出栈一次。我举个例子，比如温度数组是 [30, 40, 50, 60, 30 ,70]。遍历过程大概是：

  • i=0，栈空，0 入栈
  • i=1，temperatures[1] == 40 比栈顶 temperatures[0]大，0 出栈，记录answer[0] = 1 - 0, 然后栈空了，1 入栈
  • i=2，50 比栈顶 40 大，1 出栈，然后栈空了，2入栈
  • i=3，60 比栈顶 50 大，2 出栈，然后栈空了，3入栈
  • i = 4 , 30小于60，不进while，4也入栈，此时栈顶为[3,4],栈顶为4
  • i = 5，70 大于30 ，4 出栈 ，记录answer[ 4 ] = 4-3 =1，栈非空，循环继续，5 仍然不入栈，栈顶为3 ，answer [ 3 ] = 5- 3 = 2 ，栈空，循环结束5入栈。

你看，每个元素进来的时候可能会触发 while 循环，但是 while 每执行一次，就代表有一个元素被弹出栈了。整个遍历过程中，所有元素最多只会被弹出一次。所以 while 循环的总执行次数是 O(n)，不是每个 i 都执行 n 次。

所以总的时间复杂度 = for 循环 n 次 + while 总执行次数 n 次 = O(2n) = O(n)。

这就是所谓“均摊分析”的思路：虽然 while 看起来像嵌套循环，但它的操作次数是全局受限的，不是每个外层循环都独立内层跑一遍。

翻转二叉树

给你一棵二叉树的根节点 root ，翻转这棵二叉树，并返回其根节点。

这题其实很简单，就是简单的二叉树递归遍历然后做一些处理就行，每次遍历到左右子树再更换位置就行。

var invertTree = function (root) {
    if (!root) return null;
    let left = invertTree(root.left);
    let right = invertTree(root.right);
    root.left = right;
    root.right = left;
    return root;
};

跳跃游戏 III

这里有一个非负整数数组 arr，你最开始位于该数组的起始下标 start 处。当你位于下标 i 处时，你可以跳到 i + arr[i] 或者 i - arr[i]。

请你判断自己是否能够跳到对应元素值为 0 的 任一 下标处。

这题思路其实很简单，不管你是怎么跳，如果你跳的下一跳跳到重复的下标时，说明你这条路径是错的。最终结果到达 值为0的下标所产生的下标路径肯定是不重复的，则仿照二叉树搜索的方式做个递归就行，返回条件显而易见，这里需要注意维护visted集合，用visted.has(i) 来判断 i是否在集合中借以判断此下标是否访问过。

实际上就是BFS（广度优先搜索）

var canReact = function(arr, start, visited = new Set()) {
  // 越界检查
  if (start < 0 || start >= arr.length) return false;
  
  // 跳到已经访问过的下标，说明这条路径是错的，避免死循环
  if (visited.has(start)) return false;
  
  // 到达值为0的位置，成功，最终是并，有一个true就是成功
  if (arr[start] === 0) return true;
  
  // 标记当前下标已访问
  visited.add(start);
  
  // 递归尝试两个方向
  return canReact(arr, start + arr[start], visited) || 
         canReact(arr, start - arr[start], visited);
}

动态规划核心特征

1. 最优子结构（最明显）

当前问题的最优解可以由子问题的最优解推导出来。

判断方法：问自己“大问题的解能否由小问题的解组合得到？”

典型例子：

  • 最大正方形：dp[i][j] 由左上、上、左三个方向的最小值+1得到
  • 最长递增子序列：dp[i] 由前面所有 dp[j] 推导
  • 背包问题：当前状态由“选”或“不选”前一个物品决定

2. 重叠子问题（关键区分点）

同一个子问题会被重复计算多次。

判断方法：画出递归树，看是否有重复的节点

// 以斐波那契为例（经典DP问题）
fib(5)
├── fib(4)
│   ├── fib(3)  ← 重复
│   └── fib(2)
└── fib(3)      ← 重复
    ├── fib(2)
    └── fib(1)

对比分治法（如归并排序）：子问题不重叠，每次都是新问题

3. 无后效性（容易忽略）

某个状态一旦确定，就不受后面决策的影响。

判断方法：当前状态只关心“怎么来的”，不关心“具体怎么走的”

✅ 有后效性：走迷宫（不同路径到达同一点，会影响后续选择）
✅ 无后效性：最大正方形（只要知道边长，不需要知道具体是哪几个格子）

最大正方形

在一个由 '0' 和 '1' 组成的二维矩阵内，找到只包含 '1' 的最大正方形，并返回其面积。

这类题目显然是动态规划来解决，做一个dp[i][j] 表示以 (i, j) 为右下角的最大全1正方形的边长，显然可以写出状态转移方程：dp[i][j] = min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) + 1。写个二重循环就行。

var maximalSquare = function(matrix) {
    if (!matrix.length || !matrix[0].length) return 0;
    
    let rows = matrix.length;
    let cols = matrix[0].length;
    let maxSide = 0;
    
    // dp[i][j] 表示以 (i,j) 为右下角的最大正方形边长
    let dp = new Array(rows);
    for (let i = 0; i < rows; i++) {
        dp[i] = new Array(cols).fill(0);
    }
    
    for (let i = 0; i < rows; i++) {
        for (let j = 0; j < cols; j++) {
            if (matrix[i][j] === '1') {
                if (i === 0 || j === 0) {
                    // 第一行或第一列，最大边长只能是1
                    dp[i][j] = 1;
                } else {
                    // 状态转移方程
                    dp[i][j] = Math.min(
                        dp[i-1][j],      // 上方
                        dp[i][j-1],      // 左方
                        dp[i-1][j-1]     // 左上方
                    ) + 1;
                }
                maxSide = Math.max(maxSide, dp[i][j]);
            }
        }
    }
    
    return maxSide * maxSide;  // 返回面积
};

数组中的第K个最大元素

这题的核心在于快速排序改为快速改造，只遍历一边的数据，然后得注意对大量重复相同数据的处理，需定义equal数组。选个pivot，大的放右边小的放左边，相等放equal，相当于pivot所代表的值是第right.legth+1到第right.legth+equal.legth大的数。

function findKthLargest(nums, k) {
    if (nums.length === 1) return nums[0];

    const pivotIndex = Math.floor(nums.length / 2);
    const pivot = nums[pivotIndex];

    const left = [];
    const right = [];
    const equal = [];

    for (const num of nums) {
        if (num > pivot) {
            right.push(num);
        } else if (num < pivot) {
            left.push(num);
        } else {
            equal.push(num);
        }
    }
    //相当于pivot所代表的值是第right.legth+1到第right.legth+equal.legth大的数

    // 第 k 大在右边
    if (k <= right.length) {
        return findKthLargest(right, k);
    }

    // 第 k 大就是 pivot
    if (k <= right.length + equal.length) {
        return pivot;
    }

    // 去左边找
    return findKthLargest(
        left,
        k - right.length - equal.length
    );
}

Trie(前缀树)实现

Trie（发音类似 "try"）或者说 前缀树 是一种树形数据结构，用于高效地存储和检索字符串数据集中的键。这一数据结构有相当多的应用情景，例如自动补全和拼写检查。

请你实现 Trie 类：

  • Trie() 初始化前缀树对象。
  • void insert(String word) 向前缀树中插入字符串 word 。
  • boolean search(String word) 如果字符串 word 在前缀树中，返回 true（即，在检索之前已经插入）；否则，返回 false 。
  • boolean startsWith(String prefix) 如果之前已经插入的字符串 word 的前缀之一为 prefix ，返回 true ；否则，返回 false 。

前缀树（Trie）本质上是一种“利用字符串公共前缀来减少重复存储”的树形结构，它特别适合处理字符串查询问题，比如自动补全、搜索建议、拼写检查、敏感词过滤等。它和普通哈希表最大的区别在于：哈希表更适合精确匹配，而 Trie 天然支持前缀匹配。

Trie 的核心思想其实很简单：把字符串拆成一个一个字符，然后按字符路径存储。例如插入 "apple"，会形成：

root
 └── a
      └── p
           └── p
                └── l
                     └── e

树上的一条路径就代表一个字符串。多个字符串如果前缀相同，会共享前面的路径，例如 "apple" 和 "app" 会共用 a->p->p，这就是“前缀树”名字的来源。

在 JS 里，Trie 的每个节点本质上就是一个对象：

{
   children: {},
   isEnd: false
}

其中：children用于保存子节点；isEnd用于表示当前节点是否是一个完整单词的结尾。

很多人第一次做 Trie 时容易忽略 isEnd。例如插入 "apple" 后，搜索 "app" 时虽然路径存在，但 "app" 不一定是完整单词，所以必须通过：node.isEnd来判断。

Trie 的实现本质上是“对象嵌套对象”。例如：node.children[ch]等价于：node.children["a"]

表示当前字符对应的下一层节点。而node.children[ch] = new Trie()表示：“如果当前字符路径不存在，就创建一个新的节点”。这实际上是在动态构建一棵树。Trie 中非常关键的一句：node = node.children[ch]它的含义是：“沿着字符路径往下走”。所以 Trie 的插入、搜索、前缀匹配，本质都是：

从根节点开始，
按照字符一层层向下遍历。

例如insert("cat")会经历：root -> c -> a -> t; 每次循环： for (const ch of word)都表示正在处理一个字符。

Trie 的时间复杂度是它面试里最重要的优势。假设字符串长度为m那么：

insert  O(m)
search  O(m)
startsWith O(m)

注意这里和数据规模无关，而只和字符串长度有关。即使 Trie 里存了 100 万个单词，查找 "apple" 仍然只是遍历 5 个字符。这是 Trie 最大的优势。不过 Trie 也有明显缺点：空间消耗比较大。因为每个节点都要保存children。大量节点会带来很多对象和指针，所以 Trie 是典型的“空间换时间”。

面试里还有几个常问点。第一个是为什么 Trie 比 HashMap 更适合前缀搜索？因为哈希表只能：一次性对整个字符串做 hash它不知道字符串之间的公共前缀。而 Trie字符串路径本身就是前缀。所以：startsWith("app")只需要沿着：a -> p -> p走一遍即可。第二个常问的是：为什么 Trie 查询复杂度是 O(m)？因为 Trie 查询不需要遍历所有字符串，而是每个字符只访问一个节点。所以只与字符串长度有关。第三个是：Trie 为什么适合搜索提示？因为当前缀 "app" 找到后，Trie 已经定位到了：app 对应节点。接下来只需要 DFS 遍历它下面的所有路径，就能得到：apple，application，apply等所有自动补全结果。很多高级题目也都是 Trie 的变种，例如：

  • 单词搜索 II
  • 实现 Magic Dictionary
  • Map Sum Pairs
  • AC 自动机
  • 字典序问题

本质上都是：Trie + DFS/BFS/回溯或者Trie + 动态规划,Trie 的本质是“利用字符串公共前缀构建多叉树”，它通过“空间换时间”实现高效字符串检索，尤其适合前缀匹配、自动补全和字典类问题。以下是代码实现：

// 实现 Trie (前缀树)
class Trie {
  constructor() {
    this.children = {}; //字典，每个节点相当于可以记录无数个子节点字母
    this.isEnd = false;
  }
  insert (word){
  let node = this;

  for (const ch of word){ 
    if (!node.children[ch]){
      node.children[ch] = new Trie();
    }
    node = node.children[ch]
  }
  //单词匹配结束后结尾节点标记为一个单词结束
  node.isEnd = true;
}
}
//或者这样添加公共方法
Trie.prototype.search = function (word){
  let node = this; //根节点开始

  //按顺序遍历单词每个字母
  for(const ch of word){
    if(!node.children[ch]){
      return false;
    }

    //继续下一个字母  
    node = node.children[ch]    
  }
  //如果单词的结尾字母并不属于任何单词的结尾字母node.isEnd为false，说明这并不是完整单词。
  return node.isEnd

}

//判断是否为前缀，跟搜索差不多只是不用判断字母是否为单词结尾了
Trie.prototype.startwith = function (prefix){
  let node = this;
   for (const ch of prefix) {
        if (!node.children[ch]) {
            return false;
        }
        node = node.children[ch];
    }

    return true;

}

无环图有向图判断

你这个学期必须选修 numCourses 门课程，记为 0 到 numCourses - 1 。 在选修某些课程之前需要一些先修课程。 先修课程按数组 prerequisites 给出，其中 prerequisites[i] = [ai, bi] ，表示如果要学习课程 ai 则 必须 先学习课程 bi 。 例如，先修课程对 [0, 1] 表示：想要学习课程 0 ，你需要先完成课程 1 。 请你判断是否可能完成所有课程的学习？如果可以，返回 true ；否则，返回 false 。

这个题其实是判断有向图里有没有环，如果有环就说明有循环依赖，肯定没法完成。经典的解法是拓扑排序，可以用 DFS 或者 BFS。我用 BFS（Kahn 算法）写吧，更直观一点，思路就是：先找那些没有前置依赖的课程（入度为 0），把它们学掉，然后更新依赖这些课程的后续课程，如果后续课程的入度也变成 0 了，说明它所有的前置都学完了，就可以入队。最后如果所有课程都能入队并弹出，说明没有环。如果有环，环里的课程入度永远不会变成 0，count 就达不到 numCourses。

这个算法时间复杂度和空间复杂度都是 O(V + E)，V 是课程数，E 是先修关系的数量。

var canFinish = function(numCourses, prerequisites) {
    // 初始化入度数组，每个课程的入度是 0
    const inDegree = new Array(numCourses).fill(0);
    // 邻接表，记录每个课程的后继课程
    const adj = new Array(numCourses).fill().map(() => []);
    
    // 构建图
    for (const [ai, bi] of prerequisites) {
        inDegree[ai]++;           // ai 的入度加 1，因为它依赖 bi
        adj[bi].push(ai);         // bi 指向 ai
    }
    
    // 队列，初始把所有入度为 0 的课程放进去
    const queue = [];
    for (let i = 0; i < numCourses; i++) {
        if (inDegree[i] === 0) queue.push(i);
    }
    
    // 记录已修完的课程数
    let count = 0;
    
    while (queue.length > 0) {
        const course = queue.shift();
        count++;
        
        // 把当前课程的所有后继课程的入度减 1
        for (const next of adj[course]) {
            inDegree[next]--;
            if (inDegree[next] === 0) {
                queue.push(next);
            }
        }
    }
    
    // 如果修完的课程数等于总课程数，说明没有环
    return count === numCourses;
};

反转链表

给你单链表的头节点 head ，请你反转链表，并返回反转后的链表的头节点。

这道题很简单，首先我想到的是利用栈，将每个节点入栈，然后最后一个节点就是新节点的头指针，然后再每个节点出栈，重新进行相连，由于栈的后进先出，从而达成链表的反转:

var reverseList = function (head) {
  if (!head) return null;
  const stack = [];
  let cur = head;
  // 全部入栈
  while (cur) {
    stack.push(cur);
    cur = cur.next;
  }
  // 新头节点
  const root = stack.pop();
  cur = root;

  // 出栈重连
  while (stack.length) {
    cur.next = stack.pop();
    cur = cur.next;
  }
  // 最后断开
  cur.next = null;
  return root;
};

但是其实有更好的方法，定义两个指针，双指针原地反转，翻转指向后再将两个指针共同前进，当然要注意保存好cur.next，便于cur指针指向的节点前进:

var reverseList = function (head) {
  let prev = null;
  let cur = head;

  while (cur) {
    // 保存下一个节点
    const next = cur.next;
    cur.next = prev; // 反转指针
    // 双指针前进
    prev = cur;
    cur = next;
  }
  return prev;
};

岛屿数量

给你一个由 '1'（陆地）和 '0'（水）组成的的二维网格，请你计算网格中岛屿的数量。

岛屿总是被水包围，并且每座岛屿只能由水平方向和/或竖直方向上相邻的陆地连接形成。

这题核心是，直接循环遍历，每遍历到一个陆地，计数就加1，然后通过深度优先搜索寻找与其相连的陆地然后直接置为0防止相连陆地重复记录。

var numIslands = function(grid) {
    const rows = grid.length;
    const cols = grid[0].length;
    let count = 0;

    function dfs(i, j) {
        // 越界
        if ( i < 0 ||j < 0 ||i >= rows ||j >= cols ) {
            return; }

        // 是水域就直接返回
        if (grid[i][j] === '0') {
            return;
        }

        // 是陆地就淹掉当前陆地
        grid[i][j] = '0';

        // 四方向扩散
        dfs(i + 1, j);
        dfs(i - 1, j);
        dfs(i, j + 1);
        dfs(i, j - 1);
    }

    for (let i = 0; i < rows; i++) {
        for (let j = 0; j < cols; j++) {
            if (grid[i][j] === '1') {
                count++;
                dfs(i, j);
            }
        }
    }

    return count;
};

var majorityElement = function(nums) {

var productExceptSelf = function (nums) {
  let resfront = [];
  let resback = []; //倒着再来一个前缀乘
  let a = 1;
  let b = 1;
  for (let i = 0; i < nums.length; i++) {
    a = a * nums[i];
    resfront.push(a);
  }
  for (let j = nums.length - 1; j > -1; j--) {
    b = b * nums[j];
    resback.push(b);
  }
  let res = [];
  for (let m = 0; m < nums.length; m++) {
    if (m == 0) {
      res.push(resback[nums.length - m - 2]);
    } else if (m == nums.length - 1) {
      res.push(resfront[m - 1]);	
    } else {
      res.push(resfront[m - 1] * resback[nums.length - m - 2]);
    }
  }
  return res;
};

最小栈实现

给你一个整数数组 nums ，请你找出数组中乘积最大的非空连续 子数组（该子数组中至少包含一个数字），并返回该子数组所对应的乘积。

测试用例的答案是一个 32-位 整数。

请注意，一个只包含一个元素的数组的乘积是这个元素的值。

实际就是定义两个栈，一个存原始数据一个存排序的原始数据，push原始数据的时候，值进排序的辅助栈的时候注意与栈顶元素比大小，如果小于栈顶就push，如果大于就不放进去，维持这个单调栈。

注意如果定义this,min的话，数字是基本类型，你再let min = this.min的话，是值的拷贝，改不了原始值。而数组能改，是因为他是引用类型，局部变量拿到的也是内存地址，就能改。

var MinStack = function () {
    this.val = []; //数组（引用类型），修改它的内容不需要重新赋值， 存储的是数组的【内存地址】
    this.min = []; //辅助栈，用于维护一个单调栈
};
MinStack.prototype.push = function (val) {
    let stack = this.val; //局部变量stack 也拿到了同一个内存地址的【副本】，修改的话相当于就是修改原数组了
    let mins = this.min;
    if (mins.length == 0) {
        mins.push(val);
    } 
    if (mins[mins.length - 1] >= val) {
            mins.push(val);
        }    
    stack.push(val);
    return;
};

MinStack.prototype.pop = function () {
    let stack = this.val;
    let mins = this.min;
    let val = stack.pop();
    if (mins[mins.length - 1] == val) {
        mins.pop()
    }

};

MinStack.prototype.top = function () {
    let stack = this.val;
    return stack[stack.length - 1];
};

MinStack.prototype.getMin = function () {
    return this.min[this.min.length - 1];
};

双dp动态规划

给你一个整数数组 nums ，请你找出数组中乘积最大的非空连续 子数组（该子数组中至少包含一个数字），并返回该子数组所对应的乘积。

测试用例的答案是一个 32-位 整数。

请注意，一个只包含一个元素的数组的乘积是这个元素的值。

这题目显然可以用动态规划解决，存在最优子结构，新结构可由子结构推断出来。但是得注意构造两种dp数组，因为数组中可能包含负数，最小乘积可能乘以当下的负数会变成最大乘积。

状态转移方程:

maxProduct[i] = max(nums[i], maxProduct[i-1] * nums[i], minProduct[i-1] * nums[i])
minProduct[i] = min(nums[i], maxProduct[i-1] * nums[i], minProduct[i-1] * nums[i])

代码:

var maxProduct = function (nums) {
  let n = nums.length;
  let dpmax = [];  //
  let dpmin = []
  dpmax[0] = nums[0];
  dpmin[0] = nums[0]	
  for (let i = 1; i < n; i++) {
    dpmax[i] = Math.max(nums[i], dpmax[i - 1] * nums[i],dpmin[i-1]*nums[i]);
    dpmin[i] = Math.min(nums[i], dpmin[i - 1] * nums[i],dpmax[i-1]*nums[i])
  }
  return Math.max(...dpmax);
};

链表的归并排序

给你链表的头结点 head ，请将其按 升序 排列并返回 排序后的链表 。

​ 链表不能随机访问，不好写快排，但是链表能从中间断开很方便，则可以拆成一小个一小个的链表然后再合并，由此做归并排序（Merge Sort）：

var sortList = function(head) { //传入头节点
    // 递归终止
    if (!head || !head.next) {
        return head;
    }

    // 1. 快慢指针找中点
    let slow = head;
    let fast = head;
    let prev = null;

    while (fast && fast.next) {
        prev = slow; //注意要拿个prev记录此时已在中点的slow，不然后面遍历又往后走了一步
        slow = slow.next;
       //快指针总比慢指针的速度快两倍，快指针遍历完链表的时候慢指针刚好到链表的中点
        fast = fast.next.next;

    // 2. 从中点断开链表
    prev.next = null;

    // 3. 递归排序左右两边
    let left = sortList(head);
    let right = sortList(slow);

    // 4. 合并两个有序链表
    return merge(left, right);
};

function merge(l1, l2) {
    let dummy = new ListNode(0);
    let cur = dummy;
	//拉拉链拼接链表，每次将两个链表中较小的节点添加到尾部（cur.next = l1）,同时l1链表头指针后移，再进行遍历

            cur.next = l1;
            l1 = l1.next;
        } else {
            cur.next = l2;
            l2 = l2.next;

LRU数据结构设计

请你设计并实现一个满足 LRU (最近最少使用) 缓存 约束的数据结构。

实现 LRUCache 类：

  • LRUCache(int capacity) 以 正整数 作为容量 capacity 初始化 LRU 缓存
  • int get(int key) 如果关键字 key 存在于缓存中，则返回关键字的值，否则返回 -1 。
  • void put(int key, int value) 如果关键字 key 已经存在，则变更其数据值 value ；如果不存在，则向缓存中插入该组 key-value 。如果插入操作导致关键字数量超过 capacity ，则应该 逐出 最久未使用的关键字。

函数 get 和 put 必须以 O(1) 的平均时间复杂度运行。

此题涉及到数据结构的设计，一般来说， 要get 和 put 必须以 O(1) 的平均时间复杂度运行，显然要用哈希表（用于记录数据）和双向链表（用于快速知道谁最久未使用）来共同实现这个案例。

// 双向链表节点
class Node {
    constructor(key, value) {
        this.key = key;
        this.value = value;
        // 前驱节点
        this.prev = null;
        // 后继节点
        this.next = null;
    }
}

/**
 * @param {number} capacity
 */
var LRUCache = function(capacity) {

    // 缓存容量
    this.capacity = capacity;

    // 哈希表：key -> node
    this.map = new Map();

    // 虚拟头节点
    this.head = new Node(0, 0);

    // 虚拟尾节点
    this.tail = new Node(0, 0);
    // 初始化双向链表
    this.head.next = this.tail;
    this.tail.prev = this.head;
};

/**
 * 删除某个节点
 * node.prev <-> node <-> node.next
 * 删除后： node.prev <-> node.next
 * @param {Node} node
 */
LRUCache.prototype.removeNode = function(node) {

    node.prev.next = node.next;
    node.next.prev = node.prev;
};

/**
 * 将节点插入到头部 head <-> first 
 * 插入后：head <-> node <-> first
 * @param {Node} node
 */
LRUCache.prototype.addToHead = function(node) {

    // node 指向原来的第一个节点
    node.next = this.head.next;

    // node 前驱指向 head
    node.prev = this.head;

    // 原第一个节点 prev 指向 node
    this.head.next.prev = node;

    // head next 指向 node
    this.head.next = node;
};

/**
 * 获取缓存
 * 
 * 如果存在：
 * 1. 返回 value
 * 2. 更新为最近使用（移到头部）
 * 
 * 如果不存在返回 -1
 * 
 * @param {number} key
 * @return {number}
 */
LRUCache.prototype.get = function(key) {

    // 不存在
    if (!this.map.has(key)) {
        return -1;
    }

    // 获取节点
    let node = this.map.get(key);

    // 先删除
    this.removeNode(node);

    // 再移动到头部
    this.addToHead(node);

    return node.value;
};

/**
 * 插入缓存
 * 情况1：
 * key 已存在-> 更新 value-> 移到头部
 * 
 * 情况2：
 * key 不存在-> 新建节点 -> 插入头部 -> 判断是否超容量
 * 
 * @param {number} key
 * @param {number} value
 * @return {void}
 */
LRUCache.prototype.put = function(key, value) {

    // key 已存在
    if (this.map.has(key)) {
        // 获取节点
        let node = this.map.get(key);
        // 更新 value
        node.value = value;
        // 移除旧位置
       this.removeNode(node);
        // 放到头部
        this.addToHead(node);

    } else {
        // 创建新节点
        let node = new Node(key, value);
        // 加入哈希表
        this.map.set(key, node);
        // 插入头部
       this.addToHead(node);
        // 超出容量
        if (this.map.size > this.capacity) {
            // 获取最久未使用节点
            // tail.prev 就是链表最后一个节点
            let removeNode = this.tail.prev;
            // 从链表删除
            this.removeNode(removeNode);
            // 从 map 删除
            this.map.delete(removeNode.key);
        }
    }
};

var hasCycle = function(head) {
    if (!head || !head.next) {
        return false;
    }

    let slow = head;
    let fast = head;

    while (fast && fast.next) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow === fast) {
            return true;
        }
    }

    return false;
};

单词拆分

给你一个字符串 s 和一个字符串列表 wordDict 作为字典。如果可以利用字典中出现的一个或多个单词拼接出 s 则返回 true。

这里定义dp[i] 表示字符串s中的前 i 个字符是否能被拆分。对于当前位置 i，我们枚举一个分割点 j。把字符串拆成：s[0:j] + s[j:i]，则状态转移方程为： $$ dp[i]=\bigvee_{0\le j<i}\left(dp[j]\ \land\ s[j

]\in wordDict\right) $$

function wordBreak(s, wordDict) {
    // 用 Set 提高查找效率
    const wordSet = new Set(wordDict);
    // dp[i] 表示 s 的前 i 个字符是否可以被拼接出来
    const dp = new Array(s.length + 1).fill(false);
    // 空字符串默认可以拼接
    dp[0] = true;
    // 遍历字符串长度
    for (let i = 1; i <= s.length; i++) {
        // 枚举分割点
        for (let j = 0; j < i; j++) {
            // 前半部分可拆分 && 后半部分在字典中
            if (dp[j] && wordSet.has(s.slice(j, i))) {
                dp[i] = true;
                break; // dp[i]已经找到可行的分割方案，直接结束
            }
        }
    }

    return dp[s.length];
}

回文子串数

给你一个字符串 s ，请你统计并返回这个字符串中 回文子串 的数目。

回文字符串 是正着读和倒过来读一样的字符串。子字符串 是字符串中的由连续字符组成的一个序列。

示例 ：

输入：s = "aaa"
输出：6
解释：6个回文子串: "a", "a", "a", "aa", "aa", "aaa"

二维dp解法

这道题首先会想到动态规划，然后因为每个子串有开始节点和结束节点，因此得用二维的dp来解决，用dp[ i ] [ j ]来表示字符串从i到j这个子串是否为回文子串。 s[i...j] 是否是回文取决于两端字符是否相等，以及中间部分是否是回文，则状态转移方程为： $$ dp[i][j]=(s[i]=s[j]) ∧ (j−i<2 ∨ dp[i+1][j−1]) $$ 然后显然结果是返回 dp数组中true的个数，即各类为回文的子串的总数。

注意为什么这里要倒序遍历 i？因为dp[i][j] 的形成依赖于dp[i + 1][j - 1]，显然 i 得从大到小遍历

function countSubstrings(s) {
    const n = s.length;

    // dp[i][j] 表示 s[i...j] 是否为回文串
    const dp = Array.from({ length: n }, () =>
        new Array(n).fill(false)
    );

    let count = 0;

    // 从下往上遍历
    for (let i = n - 1; i >= 0; i--) {

        // 从左往右遍历
        for (let j = i; j < n; j++) {

            // 两端字符相等
            if (s[i] === s[j]) {

                // 长度为 1 或 2
                // 或者内部已经是回文
                if (j - i <= 1 || dp[i + 1][j - 1]) {
                    dp[i][j] = true;
                    count++;
                }
            }
        }
    }

    return count;
}

中心扩展法

由于每个回文串一定是从“中心”向两边扩展的，如果回文串的长度是奇数，则以一个字符为中心扩展，为偶数则以两个。那么我完全可以定义一个函数来计算从某个字符（或者某两个）为中心所能衍生出的回文串的格式，再遍历原字符串，就能得到所有能扩展成功的次数，即count的值。

function countSubstrings(s) {
    let count = 0;
    // 中心扩展函数
    function expand(left, right) {
        //自然记录了其本身
        while ( left >= 0 && right < s.length && s[left] === s[right] ) {
            count++;
            left--;
            right++;
        }
    }
	//只传下标进行判断即可
    for (let i = 0; i < s.length; i++) {
        // 奇数长度回文
        expand(i, i);
        // 偶数长度回文，如果s [left] != s[right]自然会返回而不再扩展
        expand(i, i + 1);
    }
    return count;
}

示例 ：

输入：nums = [100,4,200,1,3,2]
输出：4
解释：最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。

这题其实很简单，首先看示例，显然他不设置序列元素在原数组中的位置与顺序，只要有这个序列就行，自然最先想到的是给数组排序然后一个个看差值是不是1，但是这样复杂度是n log (n) ，因此得用集合的方式去重，每次遍历看集合中有没有这个元素+1这个值且没有这个元素-1，保证这个元素是数字连续序列的起始值。然后计算以这个值起始的最大长度并更新maxLen。

function longestConsecutive(nums) {
    // 去重 + O(1) 查询
    const set = new Set(nums);
    let maxLen = 0;
    for (const num of set) {
        // 只有当前数字是序列起点时才开始统计
        if (!set.has(num - 1)) {
            let currentNum = num;
            let currentLen = 1;
            // 向后不断寻找连续数字
            while (set.has(currentNum + 1)) {
                currentNum++;
                currentLen++;
            }
            maxLen = Math.max(maxLen, currentLen);
        }
    }
    return maxLen;
}

    const dp = new Array(amount + 1).fill(Infinity);

    // 金额为 0 时不需要硬币
    dp[0] = 0;

    // 遍历每个金额
    for (let i = 1; i <= amount; i++) {
        // 尝试每一种硬币
        for (const coin of coins) {

向数组中的每个整数前添加 '+' 或 '-' ，然后串联起所有整数，可以构造一个 表达式 ：

  • 例如，nums = [2, 1] ，可以在 2 之前添加 '+' ，在 1 之前添加 '-' ，然后串联起来得到表达式 "+2-1" 。

实际上这个是0/1背包问题，这题最麻烦的地方是，怎么给target这个可能为负数的计算来转为正数。

实际上对于target的每种组合，就是某些数放到正集合，某些数放到负集合。由于 nums 是非负整数数组，我们记放正集合的数的总和为M，放负集合的正数的总和为N，那么一定有：M - N = target，然后M+N一定是固定为nums的总和sum:M + N = sum。那两个算式联立一下，得到 M = (sum + target) / 2。

由于P是nums中选的数组合得到的结果，那么原题目就变成了从 nums 中选一些数，使它们的和等于 P，求方案数。

自然 ，因为每个数字只能选一次，则进行0/1背包:

var findTargetSumWays = function(nums, target) {
    let sum = 0;
    for (let num of nums) {
        sum += num;
    }
    // 整除不了，说明不可能组成
    if ((target + sum) % 2 !== 0) return 0;
    let bagSize = (target + sum) / 2;
    // target 过大
    if (bagSize < 0) return 0;
    // dp[j] 表示组成 j 的方案数
    let dp = new Array(bagSize + 1).fill(0);
    // 初始化
    dp[0] = 1;
    // 遍历物品
    for (let num of nums) {
        // 0/1 背包倒序
       for (let j = bagSize; j >= num; j--) {
            dp[j] += dp[j - num];
        }
    }
    return dp[bagSize];
};

递归回溯

给定一个不含重复数字的数组 nums ，返回其 所有可能的全排列 。你可以 按任意顺序 返回答案。

这题是寻找全排列，首先可以发现明显的树状结构，按树的不同路径来形成不同的答案，则每次遍历一个不在已选元素数组中的点作为下一个分支的选择然后进行递归，递归完了把选择的那个点去除然后继续遍历。当已选元素到达nums的长度的时候说明选完了，push进结果数组然后return，

  • 使用回溯法，每次从未使用的数字中选择一个加入当前排列
  • 当排列长度等于原数组长度时，result.push([...current]);记录结果（记住不能直接push数组，要用展开式push）
  • 通过 includes 判断数字是否已被使用（因为输入不含重复数字）
  • 每层递归后撤销选择，尝试下一个数字
  • 时间复杂度： O(n × n!)，空间复杂度：O(n)
function permute(nums) {
    const result = [];
    
    function backtrack(current) {
        // 当前排列长度等于原数组长度时，找到一个完整排列
        if (current.length === nums.length) {
            result.push([...current]);
            return;
        }
        
        // 尝试每个数字
        for (let i = 0; i < nums.length; i++) {
            // 如果当前数字已经在排列中，跳过
            if (current.includes(nums[i])) continue;
            
            // 做选择
            current.push(nums[i]);
            // 递归
            backtrack(current);   
            current.pop(); // 撤销选择便于后续循环
        }
    }
    
    backtrack([]);
    return result;
}

  let need = new Array(26).fill(0);
  let window = new Array(26).fill(0);

  // 统计 p
  for (let c of p) {
    need[c.charCodeAt() - 97]++;
  }

  let left = 0;
  let right = 0;

  let res = [];

  while (right < s.length) {
    // 加入右字符
    let r = s[right].charCodeAt() - 97;
    window[r]++;

    right++;

    // 窗口长度达到 p.length
    if (right - left === p.length) {
      // 判断是否相同
      if (isSame(need, window)) {
        res.push(left);
      }

      // 移除左字符
      let l = s[left].charCodeAt() - 97;
      window[l]--;

      left++;
    }
  }

  return res;
};

function isSame(a, b) {
  for (let i = 0; i < 26; i++) {
    if (a[i] !== b[i]) {
      return false;
    }
  }

二叉树的双重dfs

给定一个二叉树的根节点 root ，和一个整数 targetSum ，求该二叉树里节点值之和等于 targetSum 的 路径 的数目。

路径 不需要从根节点开始，也不需要在叶子节点结束，但是路径方向必须是向下的（只能从父节点到子节点）。

首先读题目，路径不需要从根节点开始，因此首先一重dfs就得遍历每一个节点作为路径起点，然后再进行二重的dfs来判断路径上维护的值是否的达到了targetSum，到达了也不能急着return，因为后续的可能有一正一负也能再次达到targetSum。

var pathSum = function(root, targetSum) { 
    let count = 0
    // 从当前节点开始统计
    function dfs(node, sum) {
        if (!node) return
        sum += node.val
        if (sum === targetSum) {
            count++
        }
        dfs(node.left, sum)
        dfs(node.right, sum)
    }	
    // 枚举所有起点
    function traverse(node) {
        if (!node) return
        dfs(node, 0)
        traverse(node.left)
        traverse(node.right)
    }
    traverse(root)
    return count
};

0/1背包

给你一个 只包含正整数 的 非空 数组 nums 。请你判断是否可以将这个数组分割成两个子集，使得两个子集的元素和相等。

这类给你数组让你划分或者让你选元素组成值的问题，实际上首先得进行关系的探讨，这个题很明显，使得两个子集的元素和相等，就是判断数组中是否能选出元素 ，他们的和为数组的总和的一半。因为数组中每个元素只能选一次，不能多选，因此是0/1背包问题而不是完全背包问题，定义dp[i]为容积i的背包所装的物品的体积，遍历每个物品，每个背包有选择装这个物品或者选择不装的权力。对于背包容积的遍历应该从大往小遍历，因为大的容积是由小的决定的，dp[j] = Math.max(dp[j], dp[j - nums[i]] + nums[i] )，如果从小往大遍历容积的话，可能小的容积dp[j - nums[i]]就已经选了加入 nums[i] 了，dp[j]再选就会出现重复选择nums[i] 的情况。

var canPartition = function (nums) {
  //转化问题为是否能选择数组中的元素组合为值volume
  let sum = nums.reduce((a, b) => a + b, 0);
  if (sum % 2 != 0) {
    return false; //无法整除2，无法均分
  }
  let target = sum / 2;
  let dp = new Array(target + 1).fill(0); //从0到volume
  for (let i = 0; i < nums.length; i++) {
    // 倒序！
    for (let j = target; j >= nums[i]; j--) {
      dp[j] = Math.max(dp[j], dp[j - nums[i]] + nums[i]);
    }
  }
  return dp[target] == target;
};

贪心

假设有打乱顺序的一群人站成一个队列，数组 people 表示队列中一些人的属性（不一定按顺序）。每个 people[i] = [hi, ki] 表示第 i 个人的身高为 hi ，前面 正好 有 ki 个身高大于或等于 hi 的人。

请你重新构造并返回输入数组 people 所表示的队列。返回的队列应该格式化为数组 queue ，其中 queue[j] = [hj, kj] 是队列中第 j 个人的属性（queue[0] 是排在队列前面的人）。

这题的核心是一个非常经典的贪心思路：先安排高的人，因为矮的人不会影响高的人前面“有多少个比自己高或相等的人”，但高的人会影响矮的人。所以我们先把高个子的位置固定下来。

对于每个人 [h, k]，含义是：他前面恰好有 k 个身高大于等于 h 的人。于是我们先按照“身高从高到低”排序，如果身高相同，再按照 k 从小到大排序。排序后，高的人会先被处理，而同高度的人里，前面人数要求少的先放。

排序代码：

people.sort((a, b) => {
    if (a[0] !== b[0]) {
        return b[0] - a[0]; //身高不等就高的在前面
    }
    return a[1] - b[1]; //身高相等就k小的在前面
});

接下来依次把每个人插入到结果数组的第 k 个位置即可：

queue.splice(k, 0, person)

为什么这样是对的？因为当前插入时，结果数组里已经全部都是比当前人高或者等高的人，所以插入到下标 k，就意味着前面恰好有 k 个满足条件的人。

完整代码如下：

var reconstructQueue = function(people) {

    people.sort((a, b) => {
        if (a[0] !== b[0]) {
            return b[0] - a[0];
        }
        return a[1] - b[1];
    });

    let queue = [];
    //插入
    for (let person of people) {
        queue.splice(person[1], 0, person); 
    }

    return queue;
};

例如：

[[7,0],[4,4],[7,1],[5,0],[6,1],[5,2]]

排序后变成：

[[7,0],[7,1],[6,1],[5,0],[5,2],[4,4]]

然后依次插入：

[7,0]
[7,0],[7,1]
[7,0],[6,1],[7,1]
[5,0],[7,0],[6,1],[7,1]
[5,0],[7,0],[5,2],[6,1],[7,1]
[5,0],[7,0],[5,2],[6,1],[4,4],[7,1]

最终得到合法队列。这题最关键的地方其实不是代码，而是想到“高个子先固定”的贪心策略。

最长公共前缀

给定一个字符串数组，返回所有字符串的最长公共前缀；如果没有公共前缀，返回空字符串 ""

这道题本质上是“横向比较字符串”。先拿第一个字符串作为公共前缀。然后不断和后面的字符串比较：

  • 如果当前字符串包含这个前缀 → 继续
  • 如果不包含 → 缩短前缀
  • 一直缩到匹配为止
function longestCommonPrefix(strs) {
  if (!strs.length) return ""

  // 默认第一个字符串为公共前缀
  let prefix = strs[0]

  for (let i = 1; i < strs.length; i++) {

    // 当前字符串不包含 prefix
    while (strs[i].indexOf(prefix) !== 0) {

      // 不断缩短 prefix
      prefix = prefix.slice(0, prefix.length - 1)

      // 已经没有公共前缀
      if (prefix === "") return ""
    }
  }

  return prefix
}
