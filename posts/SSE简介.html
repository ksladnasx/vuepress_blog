  • SSE简介

浏览器SSE（Server-Sent Events，服务器发送事件）是一种基于HTTP的技术，允许服务器主动将实时数据推送给客户端（如网页浏览器），而无需客户端反复请求。

SSE简介

SSE基本介绍

🔍 SSE的核心特性

SSE的核心工作机制是，客户端通过创建一个EventSource对象与服务器建立持久连接。服务器则通过发送具有特定Content-Type: text/event-stream头信息的响应，来保持此连接开放并持续推送数据。

其关键特性包括：

  • 单向通信：数据流只能从服务器流向客户端。客户端无法通过同一连接向服务器发送数据

  • 基于HTTP：SSE构建在标准HTTP协议之上，兼容性良好，无需像WebSocket那样引入新的协议

  • 自动重连：浏览器内置了自动重连机制，连接意外中断时会尝试重新建立连接

  • 标准消息格式：数据格式简单，通常为文本（UTF-8编码）。服务器发送的消息可包含事件类型（event）、数据本身（data）、消息ID（id）和重连时间（retry）等字段

⚡ SSE的连接数限制与解决方案

关于您提到的连接数限制，这是一个关键点：

  • 默认限制：在HTTP/1.1协议下，大多数现代浏览器（如Chrome、Firefox、Edge）对同一域名（协议+域名+端口）的并发连接数限制通常为6个。这意味着，同一个浏览器标签页（或同一浏览器实例）中，对相同域名最多只能同时建立约6个SSE连接，超出此数量的新连接会被浏览器阻塞，直到有连接被关闭

  • 限制原因：此限制是HTTP/1.1规范中的历史遗留设定，旨在防止单个客户端过度消耗服务器资源，确保网络资源公平分配。

若您的应用需要突破此限制，可考虑以下方案：

解决方案描述适用场景
升级至HTTP/2HTTP/2支持多路复用，可在单个TCP连接上并行处理多个请求和响应，从而从根本上解决连接数限制问题。长远和推荐方案，需服务器和浏览器支持，通常要求使用HTTPS。
多域名/多端口策略将SSE连接分散到不同的子域（如sse1.example.com, sse2.example.com）或不同端口号。每个子域或端口享有独立的6个连接限额。过渡方案，适用于暂无法升级HTTP/2的情况，但配置管理稍复杂。
优化连接使用及时关闭不再需要的SSE连接，或尝试将多个数据流合并到一个SSE连接中传输（通过不同事件类型区分）。基础优化，良好的连接管理习惯。

🔄 SSE与WebSocket如何选择

SSE并非万能，理解其与WebSocket的区别有助于正确选型：

特性SSE (Server-Sent Events)WebSocket
通信模式服务器->客户端（单向）客户端<->服务器（全双工）
协议基础HTTP独立的WebSocket协议（基于TCP）
复杂度相对简单，原生支持好相对复杂，常需库支持
断线重连原生支持自动重连需手动实现
数据传输仅文本（UTF-8）文本和二进制数据
典型场景实时新闻推送、股价更新、系统通知、监控仪表盘在线聊天室、实时游戏、协同编辑

简单来说，若您的应用场景主要是服务器向客户端单向推送数据，SSE因其简单性和自动重连等特性通常是更轻量、便捷的选择。若需要频繁的双向交互，则WebSocket更合适。

SSE简单示例

Vue3前端 + Node.js后端

后端关键代码 (Node.js)

const express = require('express');
const app = express();

// 存储连接的客户端
const clients = [];

// SSE端点
app.get('/sse', (req, res) => {
  res.writeHead(200, {
    'Content-Type': 'text/event-stream',
    'Cache-Control': 'no-cache',
    'Connection': 'keep-alive',
  });

  // 将客户端加入列表
  clients.push(res);

  // 发送连接成功消息
  res.write('data: 连接成功\n\n');

  // 定时发送消息
  const interval = setInterval(() => {
    res.write(`data: 服务器时间: ${new Date().toISOString()}\n\n`);
  }, 3000);

  // 客户端断开连接时清理
  req.on('close', () => {
    clearInterval(interval);
    const index = clients.indexOf(res);
    if (index !== -1) clients.splice(index, 1);
  });
});

app.listen(3000, () => console.log('Server running on port 3000'));

前端关键代码 (Vue3)

<template>
  <div>
    <h2>SSE示例</h2>
    <button @click="connect">连接</button>
    <button @click="disconnect">断开</button>
    <div>
      <p v-for="(msg, index) in messages" :key="index">{{ msg }}</p>
    </div>
  </div>
</template>

<script setup>
import { ref, onUnmounted } from 'vue';

const messages = ref([]);
let eventSource = null;

const connect = () => {
  // 创建SSE连接
  eventSource = new EventSource('http://localhost:3000/sse');
  
  eventSource.onmessage = (event) => {
    messages.value.push(event.data);
  };
  
  eventSource.onerror = (error) => {
    console.error('SSE错误:', error);
  };
};

const disconnect = () => {
  if (eventSource) {
    eventSource.close();
    eventSource = null;
  }
};

// 组件卸载时自动断开连接
onUnmounted(() => {
  disconnect();
});
</script>

运行步骤

  1. 后端：
npm install express
node server.js
  1. 前端：
npm create vue@latest
# 将上面的Vue代码放入组件
npm run dev

这个简单示例展示了SSE的核心功能：

  • 后端建立SSE连接并定时推送消息
  • 前端连接SSE并实时显示消息
  • 基本的连接/断开控制
