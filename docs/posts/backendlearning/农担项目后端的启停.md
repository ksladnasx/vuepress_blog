# 农担项目后端起停

## 启动项目

首先明白，py是有许多虚拟环境的，刚开始是在base的环境下，因此得先进入对应农担项目的后端的环境：

```bash
cd D:\Code\newfish\postwise_main\postwise\src\backend #切换到后端所在文件夹
conda env list   #查看系统有什么环境
conda activate postwise  #进入指定环境
```

然后要进行相关依赖安装（这里主要是缺少rsa） ，

```bash
python -m pip install -U pip setuptools wheel
python -m pip install coloredlogs==15.0.1
python -m pip install -r pyproject.toml -c .\constraints.txt
python -m pip install rsa  #安装相关依赖
```

当然要确保电脑安装了pandoc：

1. 浏览器下载 Windows 安装包：
   https://github.com/jgm/pandoc/releases
   选 `pandoc-xxx-windows-x86_64.msi`
2. 把 `C:\Program Files\Pandoc\` 加到 PATH
3. 关闭并重开 PowerShell
4. 验证：pandoc --version

只要这条有输出，`pypandoc` 基本就能识别。

然后启动项目：

```bash
python -m uvicorn postwise.main:app --host 0.0.0.0 --port 7860  
```



## 虚拟环境说明

不用记那么多命令，把 Conda 虚拟环境理解成 **每个项目都有自己独立的 Python 和依赖** 就行了。

例如，你电脑里只有一个 Python，如果 FastAPI 项目需要 `SQLAlchemy 2.0`，而另一个 Django 项目需要 `SQLAlchemy 1.4`，它们装在同一个环境里就会发生冲突。所以实际开发中，基本都是**一个项目对应一个虚拟环境**。

例如：

```text
base                （默认环境，尽量保持干净）
│
├── fastapi-template（学习 FastAPI）
├── dify            （运行 Dify）
├── langgraph       （学习 LangGraph）
└── pytorch         （深度学习）
```

这样每个环境互不影响，即使删掉一个环境，也不会影响其他项目。

常用操作其实只有下面几个。

查看已有环境：

```bash
conda env list
```

创建一个新环境，例如给 FastAPI 项目使用 Python 3.12：

```bash
conda create -n fastapi-template python=3.12
```

进入环境：

```bash
conda activate fastapi-template
```

成功后，终端前面会出现：

```text
(fastapi-template)
```

说明之后安装的所有依赖都会装到这个环境里，而不会影响其他项目。

退出当前环境：

```bash
conda deactivate
```

查看当前环境安装了哪些包：

```bash
pip list
```

或者：

```bash
conda list
```

如果环境不用了，先退出该环境，然后删除：

```bash
conda remove -n fastapi-template --all
```

或者：

```bash
conda env remove -n fastapi-template
```

删除后，再执行：

```bash
conda env list
```

就看不到这个环境了。

------

**对于你现在学习 FastAPI，我建议这样管理环境：**

```text
base
│
├── fastapi-template   ← 学习官方模板
├── dify               ← 以后运行 Dify
└── langgraph          ← 学习 Agent
```

每次开发前，只需要进入对应环境：

```bash
conda activate fastapi-template
```

开发完成后退出：

```bash
conda deactivate
```

这样就形成了企业开发中最常见的工作流程：**一个项目，一个虚拟环境**。
