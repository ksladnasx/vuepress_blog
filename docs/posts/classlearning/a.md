## 一、决策树（ID3算法）

### 核心思想
> **用“问问题”的方式做决策，每个问题把数据分成更纯的子集**

### 形象例子：判断是否适合相亲

```
【根节点】房、车、收入、长相？选哪个先问？
         ↓
第1问：有房吗？
   ├─ 有 → 第2问：性格好吗？
   │        ├─ 好 → 【相亲成功】
   │        └─ 不好 → 【待定】
   └─ 无 → 【放弃】
```

### ID3 的核心：信息增益

**公式**：
\[
\text{信息增益}(D, A) = \text{熵}(D) - \sum_{v=1}^{V} \frac{|D_v|}{|D|} \cdot \text{熵}(D_v)
\]

| 概念              | 公式                                      | 含义                                  |
| ----------------- | ----------------------------------------- | ------------------------------------- |
| **熵（Entropy）** | \(H(D) = -\sum_{k=1}^{K} p_k \log_2 p_k\) | 数据集的**混乱程度**（越纯 → 熵越小） |
| **条件熵**        | \(H(D\|A) = \sum \frac{                   | D_v                                   |
| **信息增益**      | \(IG = H(D) - H(D\|A)\)                   | 划分后**混乱减少了多少**              |

**ID3 的选择策略**：每次选**信息增益最大**的属性作为分裂节点

### 一个完整例子

```
数据：10个人，5个相亲成功（+），5个失败（-）

初始熵：H = -0.5*log₂0.5 - 0.5*log₂0.5 = 1（完全混乱）

【考虑“有房”属性】
有房：6人（5成功+，1失败-）→ 熵小
无房：4人（0成功，4失败）→ 熵=0（纯）

条件熵 = 6/10 * H(5/6,1/6) + 4/10 * 0 ≈ 0.65

信息增益 = 1 - 0.65 = 0.35

【考虑“性格好”属性】计算出的信息增益=0.12
→ 选“有房”作为根节点 ✅
```

### ID3 的局限与改进

| 问题             | 改进算法              |
| ---------------- | --------------------- |
| 偏向取值多的属性 | C4.5 用**信息增益比** |
| 不能处理连续值   | C4.5 做离散化         |
| 容易过拟合       | 剪枝（预剪枝/后剪枝） |
| 只能分类         | CART（可回归）        |

## 二、集成学习（元算法）

### 核心思想
> **三个臭皮匠，顶个诸葛亮** — 组合多个弱学习器，得到强学习器

### 三种主流方法

| 方法         | 策略               | 形象比喻                       | 代表算法                |
| ------------ | ------------------ | ------------------------------ | ----------------------- |
| **Bagging**  | 并行训练，投票平均 | 多个专家独立看病，投票决定     | 随机森林                |
| **Boosting** | 串行训练，关注错误 | 错题本，每次专攻上次做错的     | AdaBoost, GBDT, XGBoost |
| **Stacking** | 层级组合           | 让一群专家投票，再由总裁判决定 | 多模型融合              |

---

### 1. Bagging（并行）

**流程**：
1. 有放回采样 → 生成 N 个不同的训练集
2. 每个训练集训练一个基模型
3. 预测时：分类 → 投票；回归 → 平均

**形象例子**：
> 100个医生独立诊断同一个病人，**投票**决定最终结果

**特点**：减少方差（减少过拟合）

---

### 2. Boosting（串行）

**流程**：
1. 第1个模型训练所有数据
2. 找出第1个模型**预测错的样本**，提高它们的权重
3. 第2个模型**重点关注**这些难样本
4. 重复，最后**加权投票**

**形象例子**：
> 老师发卷子：第1次考试大家错题很多 → 把错题整理成错题本 → 第2次专攻错题 → 第3次... → 学生越来越强

**代表算法**：

| 算法         | 特点                                |
| ------------ | ----------------------------------- |
| **AdaBoost** | 调整样本权重，最后一票加权          |
| **GBDT**     | 每轮拟合上一轮的**残差**（错误）    |
| **XGBoost**  | GBDT 的工程优化版（速度快、效果好） |
| **LightGBM** | 微软出品，更快，更省内存            |

**特点**：减少偏差（提高拟合能力），但可能过拟合

---

### 3. 随机森林 = Bagging + 随机特征

**额外创新**：
- Bagging：有放回采样不同**样本**
- 随机森林：在 Bagging 基础上，每次分裂时**随机选一部分特征**（不是用全部特征）

**效果**：进一步增加多样性 → 泛化能力更强

---

### 集成学习的公式表达

**回归问题的 Bagging**：
\[
\hat{f}_{\text{bag}}(x) = \frac{1}{B} \sum_{b=1}^{B} \hat{f}_b(x)
\]

**分类问题的 AdaBoost 最终预测**：
\[
H(x) = \text{sign}\left( \sum_{t=1}^{T} \alpha_t h_t(x) \right)
\]
- \(h_t(x)\)：第 t 个弱分类器
- \(\alpha_t\)：该分类器的权重（准确率越高，权重越大）

## 三、贝叶斯分类

### 核心思想
> **用概率做决策：给定特征，计算属于每个类别的概率，选概率最大的**

### 贝叶斯定理
\[
P(y|x) = \frac{P(x|y) \cdot P(y)}{P(x)}
\]

| 符号     | 名称         | 含义                              |
| -------- | ------------ | --------------------------------- |
| \(P(y    | x)\)         | **后验概率**                      |
| \(P(x    | y)\)         | **似然**                          |
| \(P(y)\) | **先验概率** | 不看特征，类别 y 本身的概率       |
| \(P(x)\) | **证据**     | 特征 x 出现的总概率（归一化常数） |

### 贝叶斯决策规则
\[
\hat{y} = \arg\max_{y} P(y|x) = \arg\max_{y} P(x|y) \cdot P(y)
\]

---

### 朴素贝叶斯（Naive Bayes）

**“朴素”的含义**：假设所有特征**条件独立**（现实中几乎不成立，但效果出奇好）

\[
P(x_1, x_2, ..., x_n | y) = \prod_{i=1}^{n} P(x_i | y)
\]

**分类公式**：
\[
\hat{y} = \arg\max_{y} P(y) \prod_{i=1}^{n} P(x_i | y)
\]

### 形象例子：垃圾邮件分类

```
先验：P(垃圾)=0.6, P(正常)=0.4

收到邮件包含单词：“发票”、“免费”、“点击”

计算：
P(垃圾|邮件) ∝ P(垃圾) × P(“发票”|垃圾) × P(“免费”|垃圾) × P(“点击”|垃圾)
P(正常|邮件) ∝ P(正常) × P(“发票”|正常) × P(“免费”|正常) × P(“点击”|正常)

哪边大就判给哪类
```

### 各种贝叶斯变体

| 变体                 | 适用场景             | 特征分布假设     |
| -------------------- | -------------------- | ---------------- |
| **高斯朴素贝叶斯**   | 连续特征             | 正态分布         |
| **多项式朴素贝叶斯** | 文本分类（词频）     | 多项分布         |
| **伯努利朴素贝叶斯** | 文本分类（是否出现） | 0/1 分布         |
| **半朴素贝叶斯**     | 特征有小部分依赖     | 允许少量特征相关 |

## 四、三种方法对比总结

| 维度           | 决策树（ID3）       | 集成学习              | 贝叶斯分类               |
| -------------- | ------------------- | --------------------- | ------------------------ |
| **核心思想**   | 信息增益分而治之    | 多个弱模型组合        | 贝叶斯公式求后验概率最大 |
| **可解释性**   | ★★★★★（白盒）       | ★★（黑盒）            | ★★★★（概率清晰）         |
| **特征依赖**   | 自动处理            | 同基模型              | 朴素贝叶斯假设独立       |
| **数据量要求** | 中等                | 较大                  | 小到中等（可处理缺失）   |
| **过拟合风险** | 高（需剪枝）        | Boosting高/Bagging低  | 低（模型简单）           |
| **典型代表**   | ID3, C4.5, CART     | RF, XGBoost, AdaBoost | 朴素贝叶斯, TAN          |
| **输出**       | 类别                | 类别/概率             | 概率（可调节阈值）       |
| **连续值处理** | 需离散化（ID3不行） | 需基模型支持          | 高斯朴素贝叶斯可以       |

## 五、形象总结图（脑补）

```
【决策树】像一本“流程图手册”
    有房吗？→ 性格好吗？→ 收入高吗？→ 最终结论

【随机森林】像“100个专家委员会”
    每个专家有一本不同的流程图（数据+特征不同）
    最后投票决定

【Boosting】像“错题本刷题法”
    第1轮 → 做错题 → 第2轮专攻错题 → 做对新题 → 第3轮...
    每次加强薄弱环节

【朴素贝叶斯】像“根据历史统计猜答案”
    以前所有“垃圾邮件”中，80%有“发票”这个词
    那新邮件有“发票”，就很可能是垃圾
```
