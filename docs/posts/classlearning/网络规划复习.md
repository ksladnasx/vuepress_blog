# 网络规划复习

## 模块一 网络规划基础与系统生命周期

### 一、核心概念

1. **网络规划**：对用户网络应用需求进行**分析**，构思、设计出满足用户日常生产需求的**网络系统**的过程。
2. **网络系统生命周期**：网络从构思到淘汰的完整循环迭代过程，迭代动力核心是**网络应用需求的变更**。

### 二、三类生命周期模型（高频选择题考点）

|    模型    |                         阶段顺序                         |               核心特点               |                  适用场景                  |
| :--------: | :------------------------------------------------------: | :----------------------------------: | :----------------------------------------: |
| 四阶段周期 |    构思与规划 → 分析与设计 → 实施与构建 → 运行与维护     |    宏观管理、阶段可重叠、灵活性强    |          中小型项目、需求灵活多变          |
| 五阶段周期 | 需求规范 → 通信规范 → 逻辑网络设计 → 物理网络设计 → 实施 | 瀑布式流程、阶段边界清晰、变更成本高 | 网络规模较大、需求明确、需求变更较小的工程 |
| 六阶段周期 |       在五阶段基础上，**实施前增加测试与优化阶段**       |      重视质量验证、提前发现问题      |           高可靠性要求的核心网络           |

> 易错点：五阶段顺序必须牢记，逻辑设计在物理设计之前；通信规范阶段用于分析现有网络流量与瓶颈。

### 三、网络开发核心阶段与输出

1. **最关键阶段：需求规范（需求分析）阶段**，输出**需求说明书（需求规格说明书）**，是整个项目的核心依据。
2. **逻辑网络设计阶段**：完成拓扑结构、IP 地址分配、路由协议、VLAN 规划、安全策略等设计；输出逻辑设计图、地址方案、安全方案等，**不涉及具体物理设备与布线**。
3. **物理网络设计**：核心是**实现逻辑设计的具体落地部署**，包含物理拓扑、综合布线、设备选型、机房供电等；输出设备清单、施工方案。

### 四、网络设计的约束因素

- 政策约束：行业标准、监管要求、单位制度（如行业技术标准）
- 预算约束：建设与运维经费限制
- 时间约束：建设周期、上线节点要求
- 技术约束：现有技术体制、人员能力、旧设备兼容性

### 五、辅助工具

- Visio：用于绘制**网络拓扑图、流程图**，辅助系统方案理解。
- AI 辅助规划工具：解决人工调研效率低、规划精度低的问题，实现数据驱动的精准规划。

## 模块二 局域网交换与园区网基础

### 一、三层网络架构

|  层级  |                 核心功能                 |
| :----: | :--------------------------------------: |
| 核心层 | 高速数据转发、高速交换，保障可靠性与带宽 |
| 汇聚层 |   流量汇聚、策略控制、安全与 QoS 实施    |
| 接入层 |    终端用户接入、端口安全、VLAN 划分     |

### 二、VTP（VLAN 中继协议）

1. **传播条件**：VTP 信息只能在**Trunk（中继）端口**上传播。
2. 三种工作模式
   - 服务器模式（Server）：可创建 / 修改 / 删除 VLAN，接收并传播 VTP 通告，同步 VLAN 数据库
   - 客户端模式（Client）：不能修改 VLAN，接收并传播通告，被动同步 VLAN 数据库
   - 透明模式（Transparent）：可创建本地 VLAN，转发但不处理 VTP 通告，不同步数据库
3. **VTP 域**：共享相同 VLAN 数据库的交换机构成一个 VTP 管理域。

### 三、生成树协议（STP）

1. **作用**：解决交换机环路问题，实现链路冗余备份。
2. **端口四个状态**：阻塞 → 侦听 → **学习** → 转发。
3. **PortFast（快速端口）**：用于接入终端的端口，跳过侦听和学习状态，直接进入转发；仅适用于接入端口，不能用于交换机互联端口。

### 四、交换机核心配置命令（填空题高频）

```ios
! 1. 修改设备名
Switch(config)# hostname Sw1

! 2. 配置管理IP（VLAN1接口）
Sw1(config)# interface vlan 1
Sw1(config-if)# ip address 192.168.1.1 255.255.255.0
Sw1(config-if)# no shutdown

! 3. 配置默认网关
Sw1(config)# ip default-gateway 192.168.1.254

! 4. 端口设为接入模式并划入VLAN
Sw1(config)# interface range f0/1-20
Sw1(config-if-range)# switchport mode access
Sw1(config-if-range)# switchport access vlan 10

! 5. 配置快速端口
Sw1(config-if-range)# spanning-tree portfast
```

> 易错点：二层交换机的管理 IP 配置在 VLAN1 接口上，而非物理端口。

## 模块三 路由、ACL 与 NAT 技术

### 一、静态路由与默认路由

1. **静态路由格式**：`ip route 目标网段 子网掩码 下一跳地址/出接口`
2. **默认路由**：`ip route 0.0.0.0 0.0.0.0 出接口/下一跳`，用于匹配所有未知网段，通常配置在出口网关。

### 二、访问控制列表（ACL）

1. **核心作用**：对数据包进行过滤，实现访问控制、流量限制。
2. 通配符掩码（反掩码）：0 表示匹配，1 表示忽略。
   - 例：匹配 172.30.16.0~172.30.31.0/24，反掩码为`0.0.15.255`
   - 对应命令：`access-list 10 permit 172.30.16.0 0.0.15.255`
3. 扩展 ACL：可匹配源 IP、目的 IP、协议、端口号
   - 允许访问服务器 WWW 服务：`access-list 101 permit tcp any host 202.110.10.1 eq www`
   - 禁止所有 Telnet：`access-list 100 deny any any eq telnet`

### 三、NAT 网络地址转换

1. 三类 NAT 及适用场景
   - 静态 NAT：一对一固定映射，适合**对外提供服务的服务器**（Web、邮件、FTP）
   - 动态 NAT：地址池动态分配，适合内网终端批量上网
   - PAT（端口地址转换）：多对一复用公网 IP，最常用，适合大量内部个人电脑上网
2. **企业 NAT 原则**：对外服务器用静态 NAT；个人电脑用动态 NAT/PAT；不访问外网的内部服务器不做 NAT。
3. NAT 负载均衡（目的地址转换）配置要点
   - `ip nat pool 地址池名 起始IP 结束IP netmask 掩码 rotary`：`rotary`实现轮询负载均衡
   - `ip nat inside destination list 访问列表 pool 地址池名`：目的地址转换
   - 接口配置：内网口`ip nat inside`，外网口`ip nat outside`

### 四、远程访问与 PPP

1. **远程访问三类服务**：电路交换服务、分组交换服务、专线（租用线路）服务。
2. PPP 两种认证方式
   - PAP：明文传输密码，安全性低，两次握手
   - CHAP：密文挑战握手，安全性高，三次握手

## 模块四 无线网络规划（案例题高频）

### 一、无线校园网核心需求维度

覆盖范围、并发用户数、接入安全、业务带宽、建设周期、与现有网络兼容。

### 二、核心方案选型

1. **组网架构**：大规模校园网优先采用 **AC + 瘦 AP（FIT AP）** 架构，统一管理、漫游便捷、扩展性强。
2. **室外供电**：优先采用**PoE 以太网供电**；偏远点位采用光电复合缆，或太阳能供电方案。
3. **安全接入**：采用**802.1X + Portal 认证 + Radius 服务器**，绑定校园账号，实现非本校人员禁止接入。
4. **远距离点位接入**：远离核心网的区域，采用单模光纤延伸链路，或室外无线网桥点对点回传。
5. **设备选型原则**：教育场景优先选择企业级、可管理、高性价比的设备，兼顾扩展性与运维便捷性。

## 模块五 算力网络与 AI 集群网络规划（高频重点）

### 一、算力网络基础概念

1. **定义**：以算力资源为核心对象，网络承担连接与调度载体，实现算力统一感知、编排与调度，解决算力孤岛问题。
2. 与云计算、边缘计算的关系
   - 云计算：聚焦数据中心算力池化，网络仅作连接管道
   - 边缘计算：算力下沉终端侧，降低时延
   - 算力网络：是二者的延伸，实现云 - 边 - 端算力全局调度；云、边算力是算力网络的资源组成部分
3. **云网融合**：实现云与网状态共享、协同编排、联动调整，使应用 - 算力 - 网络形成一体化协同。

### 二、核心技术

1. **SRv6**：解决 “怎么走”，实现可编程路径转发，算力调度中用于灵活编排流量路径，保障业务 SLA。
2. **APN6**：解决 “识别谁”，携带业务标识，让网络感知业务类型，实现算力感知路由。
3. SDN 与 NFV
   - SDN：集中控制，提供统一路径策略编排
   - NFV：网络功能软件化，按需装配网络功能，与算力调度联动
   - 二者协同支撑算力网络的灵活调度

### 三、算力网络规划五步流程

需求分析 → 业务类型识别 → 算力量化 → 算力布局规划 → 网络承载规划

### 四、算力量化核心指标

1. 算力类：推理 QPS、卡数 / 节点数、显存、弹性峰值算力
2. 网络类：网络 I/O 带宽、时延预算、丢包率、抖动
3. 核心公式：**所需卡数 ≈ 总 QPS / 单卡可稳定支撑 QPS**

### 五、AI 集群核心通信模式

1. **All-Reduce**：数据并行训练的核心集体通信操作，聚合所有节点梯度并平均分发，保证模型参数一致。
2. **通信特征**：全节点全互联、数据量大、同步依赖强，性能由最慢节点决定。
3. **其他流量**：训练数据持续读取、Checkpoint 周期性突发写入，会与训练流量抢占带宽。

### 六、集群网络拓扑与阻塞比

1. **Leaf-Spine（Clos）拓扑优势**：结构对称、路径跳数固定、扩展标准化、运维简单，适合 AI 训练集群。
2. 阻塞比（Oversubscription Ratio）：描述上行与下行带宽的匹配关系。
   - 阻塞比越高，上行带宽越紧张，All-Reduce 越容易拥塞、性能波动越大
   - 大模型训练集群原则：优先 1:1 无阻塞～2:1 低阻塞比，不建议超过 3:1
3. **扩展方式**：扩容 Leaf 节点并全上联 Spine；带宽不足时同步扩容 Spine，维持阻塞比稳定。

### 七、无损网络技术

1. **RoCEv2**：基于以太网的 RDMA 技术，成本低、生态通用；对丢包极其敏感，需 PFC/ECN 构建无损以太网。
2. PFC/ECN 作用
   - PFC：逐跳流量控制，拥塞时暂停对应优先级流量，避免丢包
   - ECN：端到端拥塞通知，提前降速缓解拥塞
3. **规划不当风险**：PFC 死锁、拥塞雪崩、训练性能骤降、带宽利用率低。

### 八、InfiniBand vs 高性能以太网（RoCEv2）

|   维度   |               InfiniBand                |                   RoCEv2 以太网                   |
| :------: | :-------------------------------------: | :-----------------------------------------------: |
|   优势   | 原生 RDMA，时延极低、抖动极小，原生无损 | 标准以太网生态，成本低，运维门槛低，兼容混合业务  |
|   限制   |  专用硬件成本高，技术封闭，运维人才少   |           需调优 PFC/ECN，时延抖动略逊            |
| 适用场景 |      千卡以上超大规模核心训练集群       | 中小规模集群、训练推理混合、预算有限的高校 / 企业 |

### 九、拥塞雪崩防控要点

1. 拓扑：对称 Clos 架构，避免路径不均
2. 阻塞比：低阻塞比设计，从根源降低拥塞概率
3. 拥塞控制：全网统一 PFC+ECN 参数，配置死锁检测
4. 流量隔离：训练、存储、管理流量分优先级、分 VLAN，Checkpoint 错峰

## 模块六 简答题高频答题要点

### 1. 网络规划的核心内容

需求调研分析、生命周期模型选型、逻辑网络设计、物理网络设计、实施运维方案设计、成本与风险评估。

### 2. 网络系统生命周期三类模型及特点

见模块一表格，核心记忆：四阶段灵活、五阶段瀑布式、六阶段重测试。

### 3. 网络设计的约束要素

政策、预算、时间、技术四大类，每类补充 1 个例子即可。

### 4. AI 提升网络规划效率的场景

自动流量采集分析、方案自动生成与比选、网络仿真优化、算力网络智能调度。

### 5. SRv6 在算力调度中的作用

可编程路径转发、算力感知路由、简化运维、高可靠快速重路由。

### 6. All-Reduce 对网络规划的要求

高带宽、低时延低抖动、低阻塞比、完善拥塞控制、对称拓扑。

### 7. 三级算力网络架构（中心 - 区域 - 边缘）

- 边缘层：靠近终端，低时延推理、本地预处理
- 区域层：地市汇聚，中等规模推理与模型微调
- 中心层：核心数据中心，大模型训练、全局调度
- 统一编排平台实现全局算力感知与分发。

### 8. 存算分离 vs 存算融合

- 存算融合：成本低、架构简单，但流量互相干扰，需复杂 QoS 调优；适合中小规模、预算有限场景
- 存算分离：流量物理隔离，性能稳定，运维简单；但成本高；适合大规模核心训练集群

### 9. L2TP/IPSec/MPLS VPN 对比

- L2TP：二层 VPN，成本低，原生不加密，适合远程用户接入
- IPSec：三层 VPN，安全性最高，可基于公网建设，适合跨地域分支互联
- MPLS VPN：标签转发，性能高、可靠性强，成本高，适合政务 / 大型企业骨干网

## 模块七 综合题答题框架

### 1. 企业 / 校园网需求分析类

- 缺失信息：性能带宽、安全要求、现有基础、预算、扩展需求、运维能力
- 报告评价：框架完整性、内容量化程度、模块重叠度、可落地性
- 成本优化：架构精简、设备利旧、一体化选型、轻量化安全、分期建设

### 2. AI 集群规划类

1. 业务画像：先梳理规模、框架、通信模式、流量特征，再判断带宽 / 时延 / 干扰敏感型
2. 拓扑设计：Leaf-Spine 连接关系 → 规则拓扑优势 → 扩展方式与风险
3. 存储方案：二选一 → 说明理由 → 对应优化措施（隔离 / 优先级 / 错峰 / 无损调优等）

### 3. 安全域划分类

按 “接入 - 业务 - 数据 - 管理” 分层，遵循最小权限原则；每个区域说明部署内容、访问控制规则，核心是横向隔离、纵向防护、全流程审计。

### 4. 设备配置类

按 “基础配置 → 接口配置 → 业务配置 → 验证测试” 的顺序作答，命令注意格式准确，关键字不写错。

## 考前易错点速记

1. 五阶段顺序：需求→通信→逻辑→物理→实施，逻辑设计先于物理设计。
2. VTP 三种模式：服务器可改 VLAN，客户端只能同步，透明模式本地有效。
3. STP 端口状态：阻塞、侦听、学习、转发，快速端口跳过中间两步。
4. ACL 反掩码：0 匹配 1 忽略，计算网段范围用块大小。
5. NAT 方向：inside 是内网口，outside 是外网口；负载均衡用目的地址转换 + rotary。
6. 阻塞比：数值越高，上行越容易拥塞，大模型训练优先低阻塞比。
7. SRv6 管路径，APN6 管业务识别，二者功能不混淆。
8. RoCEv2 依赖以太网，必须配合 PFC/ECN 才能实现无损。
