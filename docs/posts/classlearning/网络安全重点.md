# 网络安全期末详细复习资料

## 一、概述

### 1. 网络安全和信息安全的概念与对比

- **信息安全**：信息系统中的**信息**不会因偶然或恶意原因遭受破坏、更改、泄露，信息系统能持续不间断提供信息服务。
- **网络安全**：网络系统的**硬件、软件及其系统中的数据**受到保护，不因偶然或恶意原因遭受破坏、更改、泄露，系统连续可靠正常运行，网络服务不中断。
- **对比**：信息安全覆盖**范围更广**，涵盖所有信息载体（包括单机、存储设备等）；网络安全聚焦网络环境下的安全防护，是信息安全的重要子集。

### 2. 信息安全目标

- **可用性**：信息被授权实体访问并按需使用的特性。
- **保密性**：防止信息泄露给非授权个人或实体，仅授权用户可使用。
- **完整性**：信息未经授权不能改变的特性。
- **不可抵赖性**：信息交互中，参与者不能否认曾完成的操作或承诺，分原发和接收不可抵赖。
- **可控性**：对信息的传播过程及内容具有控制能力的特性。

### 3. 网络安全的内涵及安全模型的含义和作用

- 网络安全内涵：

  - 基础理论：密钥生成、加密解密、报文摘要、鉴别机制、数字签名等。
  - 网络安全协议：保障网络传输安全的标准化规则。
  - 网络安全技术：防火墙、入侵检测、病毒防御等。
  - 主机安全技术：访问控制、主机病毒防御、主机入侵检测等。
  - 安全标准：TCSEC、CC、等级保护、ISO 27000s 等。

  

- 安全模型的含义和作用：

  - 含义：以建模方式给出解决安全问题的方法和过程。
  - 作用：明确网络环境下信息系统的组成、结构和行为；界定安全保障相关要素及相互关系；阐明信息系统行为与安全保障要素的关联。

## 二、两种安全模型

### （一）P2DR 模型

#### 组成要素：

- **策略（Policy）**：制定安全目标和规则（如访问控制策略、数据加密策略等），为整体安全防护提供核心指导框架。
- **防护（Protection）**：实施防火墙、加密技术、访问控制、漏洞修复等预防措施，从源头减少安全风险。
- **检测（Detection）**：通过入侵检测系统（IDS）、日志审计、流量分析等手段，实时监测网络和系统状态，及时发现异常行为与安全事件。
- **响应（Response）**：针对检测到的安全事件，执行事件隔离、系统恢复、漏洞修补、攻击源追踪等操作，降低损失并防止二次攻击。

#### 安全判定逻辑

- 参数定义：

  - Pt（系统防护时间）：入侵者从发起攻击到成功突破系统防护所需的时间；
  - Dt（入侵检测时间）：从入侵行为发生到系统检测到该事件的时间；
  - Rt（响应恢复时间）：检测到入侵后，系统完成事件处理、恢复正常运行的时间。

  

- 安全判定规则：

  - 若 `Pt > Dt + Rt`：系统安全，防护能力足以支撑检测与响应完成；
  - 若 `Pt < Dt + Rt`：系统不安全，入侵已成功且未及时遏制；

  

- 暴露时间：`Et = Dt + Rt - Pt`（Et 越长，系统面临的风险与损失越大）。

#### 优缺点

- 优点：

  1. 明确安全四大核心要素（策略、防护、检测、响应）及其逻辑关系，框架清晰；
  2. 体现 “动态防护” 理念，强调防护、检测、响应的实时联动与 调整；
  3. 明确技术落地路径，给出 “规划目标 → 分析风险 → 制定策略 → 选择技术” 的系统设计流程，实用性强。

  

- 缺点：

  1. 未清晰描述网络信息系统的硬件、软件、数据等核心组成与结构；
  2. 未明确系统本身特性与安全保障机制的适配关系；
  3. 忽略了 “人员” 在安全流程中的核心作用（如运维人员、安全分析师的操作）；
  4. 未体现 “人员、系统、管理” 三者有机集成的闭环运行过程。

  

### （二）IATF（信息保障技术框架）

#### 核心要素（三层结构）：

1. **人员**：安全保障的核心，负责风险分析、策略制定、安全措施落地、系统日常运维与应急处置；
2. **技术**：安全保障的基础，通过具体技术实现防护、检测、响应、恢复等安全功能（如加密、防火墙、入侵检测等）；
3. **运行**：有机集成人员、技术、系统的动态过程，包含风险持续评估、策略迭代实施、异常实时检测、系统故障恢复等全流程操作。

**生命周期**：安全保障覆盖信息系统全生命周期，形成闭环管理：`规划组织 → 开发采购 → 实施交付 → 运行维护 → 废弃`（每个阶段均嵌入安全管控措施，避免 “重建设、轻运维”）。

**安全目标**： 实现信息系统的五大核心安全属性：**保密性、完整性、可用性、可控性、不可抵赖性**（考试高频考点，需牢记）。

#### IATF划分的四大功能区域及对应安全技术（纵深防御核心）：

|    功能区域    |                         核心安全技术                         |
| :------------: | :----------------------------------------------------------: |
|  本地计算环境  | 访问控制、数字签名、主机防火墙、病毒防护、入侵防御（HIPS）、数据加密存储等 |
|    区域边界    | 网络防火墙、VPN、网络入侵检测 / 防御系统（IDS/IPS）、流量过滤、边界隔离等 |
| 网络及基础设施 | 物理环境防护、设备容错冗余、安全路由协议、网络分段、链路加密等 |
| 支撑性基础设施 | 数据备份与恢复、日志审计、安全监控、证书管理（PKI）、应急响应预案等 |

#### 纵深防御策略：

通过 “分层部署、多技术协同” 构建立体防护体系 —— 对四大功能区域分别部署针对性安全技术，形成 “边界防护 + 内部管控 + 应急兜底” 的多层屏障，避免单一防护失效导致整体安全崩溃。

#### 优缺点：

- 优点：

  1. 突出 “人员” 在安全全流程的核心作用，弥补 P2DR 模型的短板；
  2. 明确系统功能分区与对应技术，落地性强，便于实际部署；
  3. 覆盖信息系统全生命周期，实现 “事前规划、事中管控、事后恢复” 的闭环；
  4. 纵深防御策略符合复杂网络环境的安全需求，防护更全面。

  

- 缺点：

  1. 体系较复杂，涉及人员、技术、运行多维度，落地成本较高；
  2. 对组织管理能力要求高，需建立完善的流程制度支撑，小型机构适配性较弱。

  

#### 与 P2DR 模型的核心区别：

**P2DR模型为动态调整防护策略，有明确的分层理念，落地相对较简单。IATF则为纵深防御策略，重视了人员的核心作用，明确划分了四大功能区域，但是实现复杂，需同步配套的管理流程与人员培训等。**

|   对比维度   |                P2DR 模型                 |                     IATF 框架                      |
| :----------: | :--------------------------------------: | :------------------------------------------------: |
|   核心聚焦   | 技术层面的 “防护 - 检测 - 响应” 动态循环 | 人员 + 技术 + 运行的全维度协同，强调管理与技术融合 |
|   系统描述   |         未清晰界定系统组成与结构         |       明确四大功能区域，给出具体技术适配方案       |
|   人员角色   |          未突出，仅隐含在流程中          |            作为核心要素，贯穿全生命周期            |
|   防护策略   |       动态调整防护，无明确分层理念       |          纵深防御，多层级、多区域协同防护          |
| 生命周期覆盖 |        侧重系统运行阶段的安全管控        |     覆盖规划、开发、部署、运维、废弃全生命周期     |
|  落地复杂度  |          相对简单，聚焦技术实现          |       较复杂，需同步配套管理流程、人员培训等       |

  

## 三、加密算法

### 1. 基本概念

- **对称密钥体制**：加密密钥 ke 等于解密密钥 kd，优点是**加密解密效率高**，缺点是**密钥分发和保护困难**。
- **非对称密钥体制**：加密密钥 ke 不等于解密密钥 kd，无法由一个导出另一个，优点是密钥分发容易，缺点是加密解密**计算量大**。

### 2. 分组密码与流密码

- **分组密码**：将明文分成长度固定的组，每组独立加密，代表算法有 DES、AES。
- **流密码**：一次一密钥，密钥随机产生且无相关性，加密明文流得到密文流。

### 3. RSA 公开密钥加密算法（重点）

- **核心原理**：基于大素数分解的数学难题，求出两个大素数简单，分解其乘积极难。
- **密钥生成步骤：**
  1. 选择两个不同的大素数 p 和 q，计算 `n=p×q`。(不可泄漏)
  2. 计算欧拉函数 `Φ(n)=(p-1)×(q-1)`。
  3. 从 2~Φ(n)-1 中选择与 Φ(n) 互素的数作为公钥 e。
  4. 求出满足 `(e*d) mod Φ(n) = 1` 的私钥 d。
- 加密与解密：
  - 加密：c=me mod n（m 为明文，c 为密文）。
  - 解密：m=cd mod n。
- **特点**：可实现加密和数字签名，广泛应用于密钥交换、数据加密等场景。

### 4. 两种密钥体制的结合

- 加密解密效率低的非对称密钥只负责加密对策密钥的密文，用效率高的对称密钥加密数据。
- 发送端**用对称密钥加密数据**，用**接收端公钥加密对称密钥**，接收端先解密得到对称密钥，再解密数据，兼顾效率和安全性。

## 四、报文摘要算法

### 1. 数字签名的原理

- 发送者对报文做摘要运算得到报文摘要，用**私钥**对摘要加密（数字签名），将报文与签名一同发送。
- 接收者用发送者公钥解密签名得到摘要，对收到的报文重新做摘要运算，**对比两个摘要**，一致则确认报文完整且来自发送者，实现完整性检测和不可抵赖性。

### 2. 证书的构成与分发机制

- 证书构成：

  - 明文部分：用于确认公钥与用户之间绑定关系的证明（如用户名、公钥、有效期等）。
  - 密文部分（数字签名）：认证中心（CA）用**私钥**对明文的报文摘要加密后的结果。接收者用明文部分公钥解密密文得到摘要并对比摘保证完整性和不可抵赖性。

  

- 分发机制：

  - 公钥基础设施（PKI）负责证书的生成、更新、撤销和交叉认证。
  - 包含终端实体、认证中心（CA）、注册中心（RA）、证书库，CA 通过证书证明用户与公钥的绑定，上级 CA 证明下级 CA 的公钥合法性。

  

## 五、接入控制和访问控制

### 1. 身份鉴别的概念与方法

- **概念**：验证主体的真实身份与其所声称的身份是否符合的过程，需主体拥有唯一标识信息并安全证明。

- 方法（三种）：

  - 基于共享密钥：主体证明自己知道共享密钥即可。
  - 基于用户名和口令：用户证明知道授权用户对应的用户名和口令。
  - 基于证书和私钥：主体证明知道与证书中公钥对应的私钥，确认身份。

  

### 2. 终端接入 Internet 需要解决的问题

- 建立终端与接入控制设备之间的传输路径（接入网类型包括点对点网络、以太网、无线局域网）。
- 接入控制设备完成对终端的身份鉴别（通过 PAP、CHAP、TLS 等鉴别协议）。
- 动态配置终端的网络信息（如 IP 地址）。
- 动态创建终端对应的路由项，确保数据转发。

## 六、安全协议

### 1. IPSec 的结构与所含子协议的功能及工作流程

- **结构**：由 AH 协议、ESP 协议、IKE 协议、加密算法、验证算法、安全策略组成。

- 子协议功能：

  - AH 协议：提供消息源鉴别、数据完整性鉴别、防重放攻击，**不提供加密**。
  - ESP 协议：提供数据加密、消息源鉴别、数据完整性鉴别、防重放攻击。
  - IKE 协议：实现双向身份鉴别，协商加密算法、认证算法、密钥等安全参数，建立安全关联（SA）。

  

- 工作流程：

  1. 通过 IKE 协商建立 SA（安全关联，唯一标识为 SPI、目的 IP 地址、安全协议标识符）。
  2. 按照 SA 规定的安全策略，用 AH 或 ESP 封装数据。
  3. 传输过程中通过序列号和重放窗口防重放攻击。
  4. 接收端验证鉴别数据和解密，恢复原始数据。

  

### 2. HTTPS 原理与工作流程，报文交互过程

- **原理**：基于 TCP 建立 TLS 安全连接，通过 TLS 实现 Web 服务器身份鉴别、浏览器与服务器之间 HTTP 消息的保密性和完整性。

- 工作流程：

  1. 握手阶段：完成双向身份鉴别和安全参数（加密算法、MAC 算法、密钥等）协商。
  2. 数据传输阶段：HTTP 消息经 TLS 记录协议封装，加密后传输，通过 MAC 保障完整性。

  

- 报文交互过程：

  1. Client Hello：客户端发送支持的版本、随机数、密码算法列表等。
  2. Server Hello：服务器回应支持的最高版本、随机数、选定的密码算法等。
  3. 服务器发送证书，证明自身身份。
  4. 密钥交换：客户端验证证书后，发送密钥交换信息（如预主密钥）。
  5. 双方基于随机数和预主密钥计算主密钥，衍生会话密钥。
  6. Change Cipher Spec：通知对方开始使用新协商的安全参数。
  7. Finished：双方发送完成握手的确认消息，握手结束。
  8. 传输 HTTP 消息：消息经加密和 MAC 计算后传输。

  

## 九、互联网安全技术

### NAT （网络地址转换）的作用和应用

- **作用**：解决 IP 地址短缺问题，将内部网络的私有 IP 地址转换为外部网络的公有 IP 地址，对外屏蔽内网结构，提供弱安全性。

- 应用：

  - 静态 NAT：内部本地地址与内部全球地址永久映射，**允许外部访问内部特定主机。**
  - 动态 NAT：会话开始时动态分配公有 IP，会话结束后释放。
  - PAT（端口多路复用）：多个内部本地地址映射到一个公有 IP 的不同端口，是最常用的 NAT 类型。
  
  

## 十、虚拟专用网 VPN

### 1. 第三层隧道与 IPSec 的原理及工作流程

- **原理**：第三层隧道用通用路由封装（GRE）将私有 IP 分组封装在公有 IP 分组中传输，IPSec 提供双向身份鉴别、数据保密性和完整性保障。

- 工作流程：

  1. 发送端将私有 IP 分组用 GRE 封装，添加 GRE 头。
  2. 对封装后的分组用 IPSec ESP 协议加密和添加鉴别数据。
  3. 封装成公有 IP 分组，通过 Internet 传输。
  4. 接收端解密和验证鉴别数据，**剥离** GRE 头，恢复原始私有 IP 分组。

  

### 2. 第二层隧道与 IPSec 的原理及工作流程

- **原理**：第二层隧道用 L2TP (第二层隧道协议) 建立隧道，封装 PPP 帧传输，IPSec 保障隧道中数据的安全。

- 接入流程：建立 L2TP 隧道→PPP 链路→身份鉴别→分配私有 IP→访问内网

- 工作流程：

  1. 建立控制连接：协商隧道参数和身份鉴别。
  2. 建立第二层隧道：协商会话标识等。
  3. 终端与接入控制设备建立基于隧道的 PPP 链路，完成身份鉴别和 IP 地址分配。
  4. PPP 帧经 L2TP 封装后，用 IPSec 加密传输。
  5. 接收端解密和验证，剥离封装，恢复 PPP 帧。
  
  

## 十一、防火墙

### 三种防火墙的安全机制及优缺点对比

|    类型    |                           安全机制                           |                             优点                             |                             缺点                             |
| :--------: | :----------------------------------------------------------: | :----------------------------------------------------------: | :----------------------------------------------------------: |
| 分组过滤器 |   基于源 IP、目的 IP、端口号、协议类型等规则，过滤 IP 分组   |   处理速度快、效率高，对网络性能影响小，对源和目的主机透明   | 不能检测应用层内容，无法防御基于应用层的攻击，无状态分组过滤器难以防御复杂攻击 |
| 电路层代理 | 终端先与代理建立 TCP 连接 / UDP 会话，代理鉴别身份后与服务器建立连接，绑定两个连接 |        实现基于用户的访问控制，能检测传输层字段正确性        |      不检测应用层内容，对源主机不透明，需配置代理客户端      |
| 应用层网关 |    检测应用层消息首部和消息体，验证协议规范性、攻击特征等    | 能防御针对特定应用服务器的攻击（如 SQL 注入、XSS），提供精细化安全防护 |  处理效率低，对网络性能影响较大，需针对不同应用定制检测规则  |

## 十四、计算机安全技术

### 1. 计算机安全技术概念

- 保护计算机系统的硬件、软件和数据，防止遭受病毒、黑客攻击 、越权访问等安全威胁，保障系统正常运行的技术总称，包括病毒防御技术、个人防火墙、主机入侵检测系统、访问控制等。

### 2. 访问控制模型概念

- 定义主体（用户、进程等）对客体（文件、资源等）的访问规则和权限分配方式的模型，核心是保障授权用户获取所需资源，拒绝非授权用户非法访问，常见模型有**自主访问控制模型、强制访问控制模型、基于角色的访问控制模型**等。

## 名词解释重点提示
